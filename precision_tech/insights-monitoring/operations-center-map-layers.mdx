## Overview

**Map Layers** in the John Deere **Operations Center** help users understand their operation on a **field-by-field** basis. Layer types include management zones, drone and satellite imagery, topographic and soil maps, prescriptions, financial views, and more. You can set **legend** ranges, **extent**, **metadata**, and Markdown text so growers know your tool produced the layer.

**Map Layer Summaries** group layers for a field and carry high-level context (Markdown supported). Each **Map Layer** represents **one** image (or one processed upload such as a shapefile). Map layers appear in **Field Analyzer** on the web and in **Operations Center** and **Connect Mobile** apps (Android and iOS).

## Getting started

<Steps>
  <Step title="1. Create a Map Layer Summary">
    **POST** `/platform/organizations/{orgId}/fields/{id}/mapLayerSummaries` with **`Content-Type: application/vnd.deere.axiom.v3+json`**. Use the **Location** response header (or links in the body) for the new summary URI in the next steps.
  </Step>
  <Step title="2. Create a Map Layer">
    **POST** `/platform/mapLayerSummaries/{id}/mapLayers` with the summary id from step 1. Save the Map Layer **id** and **Location** header as needed.
  </Step>
  <Step title="3. Create a File Resource">
    **POST** `/platform/mapLayers/{id}/fileResources` to reserve storage. Request returns links for upload and for checking status (**targetResource**).
  </Step>
  <Step title="4. Upload the File Resource">
    **PUT** `/platform/fileResources/{id}` with a valid **Content-Type** (for example **application/octet-stream** or **application/zip** for a shapefile). Upload **PNG** or **ZIP** shapefile as supported. Poll the File Resource (or **targetResource** link) until processing completes. Initial layer status is often **QUEUED**, then **VALID** or **INVALID**.
  </Step>
</Steps>

## Authentication (OAuth 2.0)

John Deere **Precision Tech** platform APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched callbacks can yield **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize and exchange the code">
    Request only the **scopes** your app needs. Access tokens expire after **12 hours**. Request **`offline_access`** if you need a **refresh token** (unused refresh tokens expire after **365 days**).
  </Step>
  <Step title="Enable organization access">
    After token exchange, call **`GET`** `https://sandboxapi.deere.com/platform/organizations`. If a link with **`rel=connections`** appears, redirect the user once per session to complete organization selection. Avoid redirect loops.
  </Step>
  <Step title="Call the Map Layers API">
    Send **`Authorization: Bearer {token}`** on every request. Use **`ag1`**, **`ag2`**, and **`ag3`** as required per endpoint.
  </Step>
</Steps>

### Scopes used by this API

| Scope | Typical use |
| --- | --- |
| `ag1` | List/read Map Layers, list file resources, get file resource metadata or download, extract PNG via **GET** `/platform/mapLayers/{id}`. |
| `ag2` | List Map Layer Summaries; read a Map Layer Summary. |
| `ag3` | Create/delete summaries and layers; create/delete file resources; upload files. |
| `offline_access` | Refresh tokens (optional). |

## File Resource MIME types

| Type | Description |
| --- | --- |
| `image/png` | Stays **PNG** for use in Operations Center. |
| `image/tif` | Single-band GeoTIFF; converted to **PNG** for Operations Center. |
| `image/tiff` | Same as **tif**. |
| `application/zip` | **Shapefile** (polygon only in product); converted to **PNG**. Include a **`zone_color`** column in the **.dbf** for custom colors (8-bit RGB per row; default white `(255, 255, 255)`). Other geometry types must be uploaded as **PNG** first. |

## Map Layer events (Data Subscription Service)

When a **File Resource** is uploaded, status is typically **QUEUED**, then updated when processing finishes. Subscribe to **Map Layer** events via the [Data Subscription Service](/precision_tech/application/data-subscription-service). Prefer subscriptions scoped to a **single** `orgId` (you cannot change org lists after create; losing access to any org can terminate the subscription).

### Event statuses

| Status | Description |
| --- | --- |
| `QUEUED` | Initial state after requesting a Map Layer image. |
| `VALID` | Image processed and associated with the Map Layer. |
| `INVALID` | Processing failed. |
| `NO_FILE_RESOURCE` | File resource missing in the system. |

### Example MetaDataFilter fields

| Field | Required | Example | Description |
| --- | --- | --- | --- |
| `orgId` | Yes | `"123456"` | Organization id. |
| `id` | No | UUID | Map Layer id. |
| `fieldId` | No | UUID | Field id. |
| `summaryId` | No | UUID | Map Layer Summary id. |
| `type` | No | `PRESCRIPTION` | Map layer type. |
| `status` | No | `VALID` | Image status. |
| `action` | No | `CREATED` | Action performed. |

## Compression

Send **`Accept-Encoding: gzip`** when you want gzip-compressed responses (where supported).

## Paging

List responses use **CollectionPage** (**`total`**, **`links`**, **`values`**). Default page size is **10**. Use **`itemLimit`** (multiples of **10**, up to **100**) and **`pageOffset`** to page through results.
