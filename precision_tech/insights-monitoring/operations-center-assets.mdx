## Overview

An **Asset** is an IoT device that can appear on the **John Deere Operations Center run map**. The API is flexible: you can surface readings for many device types. Each time you update data for the device, customers see the new values and **last updated** time. For movement and history, post **asset locations** with newer timestamps‚Äîusers can review **location history** as the asset moves.

<CardGroup cols={2}>
  <Card title="Run map visibility" icon="map">
    Assets show on the run map alongside equipment, machines, and boundaries‚Äîusers can open cards and detail dialogs for contributed assets.

### Example catalog mappings

These examples illustrate how **assetSubType** selects the icon in Operations Center (see **GET** `/platform/assetCatalog` for the full list).

| Asset | Example `assetSubType` | Notes |
| --- | --- | --- |
| Smart barn / high population barn | `SMART_BARN` | Barns with sensors |
| Technician truck | (equipment tank / vehicle types from catalog) | Service vehicles |
| Nurse wagon | `NURSE_WAGON` | Product support wagons |
| Grain bin | `GRAIN_BIN` | Connected bins |
| Irrigation pivot | `IRRIGATION_PIVOT` | Pivot status and position |

### Map card and Markdown

The **map card** and **details** dialog support a **subset of Markdown**, including **links** to external sites. For **measurement** rows on an asset location, only **links** in the **name** field are supported (see **Measurement data** below).

## Endpoints overview

| Method | Path | Purpose |
| --- | --- | --- |
| **GET** | `/platform/assetCatalog` | Asset catalog (categories, types, subtypes, icons). |
| **GET** | `/platform/organizations/{orgId}/assets` | All assets for an organization. |
| **POST** | `/platform/organizations/{orgId}/assets` | Create an asset (returns unique id). |
| **GET** | `/platform/assets/{assetId}` | Get one asset. |
| **PUT** | `/platform/assets/{assetId}` | Update an asset (not category/type/subtype). |
| **DELETE** | `/platform/assets/{assetId}` | Delete an asset. |
| **GET** | `/platform/assets/{assetId}/locations` | Location history; optional **startDate** / **endDate**. |
| **POST** | `/platform/assets/{assetId}/locations` | Append a new location / measurements. |

## Getting started

<Steps>
  <Step title="1. Request the asset catalog">
    **GET** `/platform/assetCatalog` with **`Accept: application/vnd.deere.axiom.v3+json`** to see allowed **assetCategory**, **assetType**, and **assetSubType** values. **assetSubType** controls the icon. Use **OTHER** when no listed subtype fits.
  </Step>
  <Step title="2. Create an asset">
    **POST** `/platform/organizations/{orgId}/assets` with a valid combination from the catalog. Save the returned **asset** id from the response or **Location** header.
  </Step>
  <Step title="3. Assign a location">
    **POST** `/platform/assets/{assetId}/locations` with geometry and timestamp. Without at least one location, the asset **does not** show on the run map.
  </Step>
  <Step title="4. Update over time">
    Post **new** asset locations with a **newer** timestamp to move the asset or refresh **measurementData**. You **cannot** update an existing location row; duplicate timestamps return **409 Conflict**. Fractional seconds are not distinguished (see below).
  </Step>
</Steps>

## Authentication (OAuth 2.0)

John Deere **Precision Tech** platform APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs** (for example `http://localhost:9090/callback`); mismatched callbacks can yield **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize and exchange the code">
    Send the user to authorize with only the **scopes** your app needs. Exchange the code at the token endpoint. Access tokens expire after **12 hours**. Request **`offline_access`** if you need a **refresh token** (unused refresh tokens expire after **365 days**).
  </Step>
  <Step title="Enable organization access">
    After token exchange, call **`GET`** `https://sandboxapi.deere.com/platform/organizations`. If a link with **`rel=connections`** appears, redirect the user once per session to complete organization selection. Avoid redirect loops.
  </Step>
  <Step title="Call the Assets API">
    Send **`Authorization: Bearer {token}`** on every request. Use **`eq1`** for read operations and **`eq2`** for create, update, delete, and location posts.
  </Step>
</Steps>

### Scopes used by this API

| Scope | Typical use |
| --- | --- |
| `eq1` | View asset catalog, list/get assets, list locations. |
| `eq2` | Create, update, delete assets; create asset locations. |
| `offline_access` | Refresh tokens (optional). |

## Compression

Send **`Accept-Encoding: gzip`** when you want gzip-compressed responses (where supported).

---

## Paging

List responses use the **CollectionPage** pattern: **`total`**, **`links`** (e.g. **nextPage** / **previousPage**), and **`values`**. Default page size is **10**. Use **`itemLimit`** (multiples of **10**, up to **100**) and **`pageOffset`** to page through results.

---

## eTags

**GET** `/platform/assets/{assetId}/locations` **does not** support John Deere‚Äôs **`x-deere-signature`** eTag optimization. Other list APIs may support eTags as described in platform documentation.

### Create a new asset

Creates an asset. Use **GET** `/platform/assetCatalog` for all valid **assetCategory**, **assetType**, and **assetSubType** combinations.

```text
POST /platform/organizations/{orgId}/assets
```

<ParamField path="orgId" type="string" required>
  Operations Center organization id.
</ParamField>

#### Body (conceptual)

<ResponseField name="assetCategory" type="string">
  From the asset catalog (for example **DEVICE**, **EQUIPMENT**).
</ResponseField>

<ResponseField name="assetType" type="string">
  From the asset catalog (for example **SENSOR**, **OTHER**).
</ResponseField>

<ResponseField name="assetSubType" type="string">
  From the asset catalog; drives the map icon (for example **SMART_BARN**, **OTHER**).
</ResponseField>

**OAuth scope:** `eq2`

<CodeGroup>

  --header 'Accept: application/vnd.deere.axiom.v3+json' \
  --header 'Content-Type: application/vnd.deere.axiom.v3+json' \

### Create new asset location

Appends a location record. Use a **new** timestamp for each post. **409 Conflict** if a location with the same timestamp already exists. **measurementData** **name** supports Markdown **links only**.

Asset locations **do not** honor fractional seconds: `2019-01-01T12:34:56.900Z` and `2019-01-01T12:34:56Z` are treated as the same instant for conflict detection.

```text
POST /platform/assets/{assetId}/locations
```

<ParamField path="assetId" type="string" required>
  Asset id.
</ParamField>

**OAuth scope:** `eq2`

<CodeGroup>

```bash cURL
curl --request POST \
  --url "https://sandboxapi.deere.com/platform/assets/a1b2c3d4-e5f6-7890-abcd-ef1234567890/locations" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.axiom.v3+json' \
  --header 'Content-Type: application/vnd.deere.axiom.v3+json' \
  --data '{"@type":"AssetLocation","timestamp":"2024-06-15T14:30:00Z","geometry":{"type":"Point","coordinates":[-93.6122,41.5234]},"measurementData":[{"name":"Humidity","value":"61","unit":"percent"}]}'
```

</CodeGroup>
