## Overview

The **Hours of Operation** API returns **durations** for which a machineā€™s **engine** was **on** or **off** during a **specified period**. You can filter by **UTC** **`startDate`** / **`endDate`**, optionally request a single **detailed state** dimension (PTO, utilization, autonomy, and similar), and inspect **`engineState`** on each segment. Each response includes **`links`** (including a **`machine`** link). **When the terminal is powered off, hours of operation are not recorded.** This endpoint **does not** support **`x-deere-signature`** / eTags.

## Getting started

1. **Authenticate** ā€” Complete OAuth 2.0 authorization code flow and organization connection (see below).
2. **Resolve `principalId`** ā€” Use the machine **principal id** from **`GET`** `/platform` discovery flows or equipment/machine resources your user can access.
3. **Call hours of operation** ā€” **`GET`** `/platform/machines/{principalId}/hoursOfOperation` with **`Accept: application/vnd.deere.axiom.v3+json`**, and optional **`startDate`** / **`endDate`** (UTC).

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs** (for example `http://localhost:9090/callback`); mismatched callbacks can yield **400 Bad Redirect**.

## Engine state codes

| Code | Meaning |
| --- | --- |

| `1` | The power-on key is **ON**. |
| `2` | The terminal is **SLEEPING**. |

## Endpoints

### Get machine hours of operation

Returns duration segments for which the engine was on or off during the requested window, with **`machine`** links. **Does not** support eTags.

```text
GET /platform/machines/{principalId}/hoursOfOperation
```

<ParamField path="principalId" type="string" required>
  **Principal id** of the machine or equipment.
</ParamField>

<ParamField query="organizationId" type="string">
  Organization id of the machine/equipment when scoping to a specific organization context.
</ParamField>

<ParamField query="startDate" type="string">
  **UTC** timestamp ā€” include segments **starting from** this instant (for example `2013-04-29T00:00:00Z`).
</ParamField>

<ParamField query="endDate" type="string">
  **UTC** timestamp ā€” include segments **before** this instant (for example `2013-04-30T23:30:00Z`).
</ParamField>

<ParamField query="detailedState" type="string">
  Send **one** value per request: **RearPTOEngagement**, **GenericMachineUtilization**, **GenericEngineUtilization**, **RoadbuildingMachineState**, **AutonomyMachineState**, or **FrontWheelDriveActuatorState**.
</ParamField>

<ParamField query="summarizeDuration" type="string">
  For supported detailed states, merges consecutive **EngagedStates** when closer than this duration (see product documentation for units and behavior).
</ParamField>

<ParamField header="Accept" type="string" required>
  Use **`application/vnd.deere.axiom.v3+json`**.
</ParamField>

**OAuth scope:** `eq1`

#### Example response shape

Responses are a **CollectionPage** with **`total`**, **`links`** (often **`self`**), and **`values`**. Each value includes **`startDate`**, **`endDate`**, **`engineState`**, optional **`detailedState`**, and **`links`** (including **`machine`**).

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://sandboxapi.deere.com/platform/machines/5432/hoursOfOperation?organizationId=98765&startDate=2013-04-29T00:00:00Z&endDate=2013-04-30T23:30:00Z&detailedState=FrontWheelDriveActuatorState&summarizeDuration=5" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.axiom.v3+json' \
  --header 'Accept-Encoding: gzip'
```

</CodeGroup>
