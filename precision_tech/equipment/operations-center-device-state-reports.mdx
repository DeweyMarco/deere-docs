## Overview

The **Machine Device State Reports** API returns **call-in history** from the machine’s **telematics terminal** for a time range. Each **device state report (DSR)** reflects the state of the terminal at a point in time: engine and terminal power, signal strength (RSSI), GPS and network diagnostics, Wi‑Fi and cellular details, antenna types, battery, boot metadata, and related fields. Reports are **terminal-scoped**—one DSR per terminal request in a single-terminal call; multi-terminal flows may return two or more DSRs per request.

### Equipment concepts

- **Implement** — Non–self-propelled equipment paired with a machine (planter, corn head, spray boom, and similar). Implements with modems may be modeled as **machine** resources today; that may change in the future.
- **EquipmentType** vs **EquipmentApexType** vs **Category** — Prefer **EquipmentType** for broad classification. **EquipmentApexType** supports agriculture displays. **Category** is legacy and may be deprecated.

## Getting started

1. **Authenticate** — Complete OAuth 2.0 authorization code flow and organization connection (see below).
2. **Resolve `principalId`** — Use the machine **principal id** from **`GET`** `/platform` discovery flows or equipment/machine resources your user can access.
3. **Call DSRs** — **`GET`** `/platform/machines/{principalId}/deviceStateReports` with **`Accept: application/vnd.deere.axiom.v3+json`** and optional **`startDate`** / **`endDate`**.

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs** (for example `http://localhost:9090/callback`); mismatched callbacks can yield **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Send the user to authorize">
    Use `response_type=code`, required **scopes** (include **`eq1`** for this API), **client_id**, **state**, and a **URL-encoded** `redirect_uri` that matches your app registration. Request only the scopes your app needs.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** the token endpoint with `grant_type=authorization_code`, the code, redirect URI, and client credentials. Access tokens expire after **12 hours**. Request **`offline_access`** if you need a **refresh token** (refresh tokens expire after **365 days** without use).
  </Step>
  <Step title="Enable organization access">
    After token exchange, call **`GET`** `https://sandboxapi.deere.com/platform/organizations` (or production). If a link with **`rel=connections`** appears, redirect the user once per session to complete organization selection. Avoid redirect loops.
  </Step>
  <Step title="Call the Device State Reports API">
    Send **`Authorization: Bearer {token}`** on every request.
  </Step>
</Steps>

## Endpoints

### List terminal device state reports

Returns **device state reports** for the machine identified by **`principalId`**. Information is collected from the **machine terminal**. Each DSR may link to **`machine`** (omit if the terminal is not linked to a machine) and **`terminal`**. This API **does not** support eTags.

```text
GET /platform/machines/{principalId}/deviceStateReports
```

<ParamField path="principalId" type="string" required>
  Machine **principal id** (from platform machine or equipment resources the user can access).
</ParamField>

<ParamField query="startDate" type="string">
  Range start (**ISO 8601** UTC). See **Date range defaults** when omitted.
</ParamField>

<ParamField query="endDate" type="string">
  Range end (**ISO 8601** UTC). See **Date range defaults** when omitted.
</ParamField>

<ParamField query="itemLimit" type="integer">
  Page size: multiples of **10**, max **100**.
</ParamField>

<ParamField query="pageOffset" type="integer">
  Offset aligned with **`itemLimit`**.
</ParamField>

<ParamField header="Accept" type="string" required>
  Use **`application/vnd.deere.axiom.v3+json`**.
</ParamField>

**OAuth scope:** `eq1`

#### Response content

Reports may include attributes such as **engine state**, **terminal power state**, **model state**, **RSSI**, local information, **GPS** state/error, **Wi‑Fi** info/error, **GSM/Wi‑Fi antenna type**, **Wi‑Fi SSID**, **battery voltage**, **last boot type**, **last boot timestamp**, and **vehicle power state**—exact fields depend on terminal and firmware.

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://sandboxapi.deere.com/platform/machines/12345/deviceStateReports?startDate=2014-01-01T00:00:00.000Z&endDate=2014-01-28T15:59:00.000Z&itemLimit=50&pageOffset=0" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.axiom.v3+json' \
  --header 'Accept-Encoding: gzip'
```

</CodeGroup>

<ResponseField name="200 OK" type="success">
