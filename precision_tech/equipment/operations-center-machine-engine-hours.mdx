## Overview

The **Machine Engine Hours** API returns the **last reported** number of **engine hours** recorded for a machine. Each report includes a **timestamp**, the **source** of the meter reading (telematics path), and typically a **`machine`** link so you can load the machine from `/platform`. This endpoint **does not** support John Deere’s **`x-deere-signature`** / eTag flow—use polling or other strategies if you need change detection.

### Equipment concepts

- **Implement** — Non–self-propelled equipment paired with a machine (planter, corn head, spray boom, and similar). Implements with modems may appear as **machine** resources today; future changes may model them separately.
- **EquipmentType** vs **EquipmentApexType** vs **Category** — Prefer **EquipmentType** for broad equipment classification. **EquipmentApexType** supports agriculture displays. **Category** is legacy and may be deprecated.

## Getting started

1. **Authenticate** — Complete OAuth 2.0 authorization code flow and organization connection (see below).
2. **Resolve `principalId`** — Use the machine **principal id** from **`GET`** `/platform` discovery flows or equipment/machine resources your user can access.
3. **Call engine hours** — **`GET`** `/platform/machines/{principalId}/engineHours` with **`Accept: application/vnd.deere.axiom.v3+json`**.

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs** (for example `http://localhost:9090/callback`); mismatched callbacks can yield **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Send the user to authorize">
    Use `response_type=code`, required **scopes** (include **`eq1`** for this API), **client_id**, **state**, and a **URL-encoded** `redirect_uri` that matches your app registration. Request only the scopes your app needs.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** the token endpoint with `grant_type=authorization_code`, the code, redirect URI, and client credentials. Access tokens expire after **12 hours**. Request **`offline_access`** if you need a **refresh token** (refresh tokens expire after **365 days** without use).
  </Step>
  <Step title="Enable organization access">
    After token exchange, call **`GET`** `https://sandboxapi.deere.com/platform/organizations` (or production). If a link with **`rel=connections`** appears, redirect the user once per session to complete organization selection. Avoid redirect loops.
  </Step>
  <Step title="Call the Engine Hours API">
    Send **`Authorization: Bearer {token}`** on every request.
  </Step>
</Steps>

## Sources and definitions

Meter readings are attributed to a **source** code indicating the telematics path:

| Code | Meaning |
| --- | --- |
| **TM** | Meter readings received from an **MTG** or **UTG**. |
| **MIG** | Meter readings received from an **MIG** (Machine Information Gateway). |
| **CI** | Meter readings received from a **GTT** (GlobalTracs Terminal). |

## Endpoints

### Get machine engine hours

Returns the last reported engine hours for the machine identified by **`principalId`**. This API **does not** support eTags.

```text
GET /platform/machines/{principalId}/engineHours
```

<ParamField path="principalId" type="string" required>
  Machine **principal id** (from platform machine or equipment resources the user can access).
</ParamField>

<ParamField header="Accept" type="string" required>
  Use **`application/vnd.deere.axiom.v3+json`**.
</ParamField>

**OAuth scope:** `eq1`

#### Example response shape

Responses are typically a **CollectionPage** with **`total`**, **`links`**, and **`values`**. Each value includes reading details and **`links`** (for example **`machine`**).

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://sandboxapi.deere.com/platform/machines/12345/engineHours" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.axiom.v3+json' \
  --header 'Accept-Encoding: gzip'
```

</CodeGroup>

#### Responses

<ResponseField name="200 OK" type="success">
  Engine hours collection (**CollectionPage** or equivalent list structure for this API version).
</ResponseField>

<ResponseField name="401 Unauthorized" type="error">
  Invalid or expired token, or missing API license—refresh the token or verify application entitlements.
</ResponseField>

<ResponseField name="403 Forbidden" type="error">
  User or client cannot access the machine, or token lacks **`eq1`**. For organization or terms issues, check **X-Deere-Warning** and **X-Deere-Terms-Location** when present.
</ResponseField>

<ResponseField name="404 Not Found" type="error">
  **`principalId`** is not valid for the caller’s current accessible resources.
</ResponseField>
