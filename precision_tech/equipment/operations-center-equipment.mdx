## Overview

The **Operations Center Equipment API** exposes **machines**, **implements**, and **devices**: hardware used for work, optionally paired with other equipment and telematic components. Use it to read equipment your users can access, create third-party managed equipment, and look up **make**, **ISG type**, and **model** ids from John Deere’s reference data before you post equipment or measurements.

### Concepts

- **Machine** — Motorized equipment (self-propelled or towed); may pair with implements.
- **Implement** — Work equipment that pairs with a machine (planter, corn head, boom, and similar).
- **Device** — Software-driven components (modem, display, receiver); details may appear when paired.

### Equipment make, type, and model

Responses from the reference endpoints supply **ids** for makes, **ISG types**, and **models** for use in **POST** `/organizations/{organizationId}/equipment`. For uploading measurements for third-party equipment, see the **Equipment Measurements** API in Dev Docs.

## Getting started

1. **Resolve model ids** — **GET** `/equipmentMakes` → **GET** `/equipmentMakes/{id}/equipmentISGTypes` → **GET** `/equipmentMakes/{id}/equipmentISGTypes/{id}/equipmentModels`. Or **GET** `/equipmentModels?equipmentModelName=…&embed=make,isgType`.
2. **Create equipment** — **POST** `/organizations/{orgId}/equipment` with `@type` **Machine** or **Implement**, `name`, optional `serialNumber`, and `model` ids.
3. **Send measurements** — Use the **Equipment Measurements (POST)** API as documented separately.

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs** (for example `http://localhost:9090/callback`); mismatched or unregistered callbacks can produce **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Send the user to authorize">
    Use `response_type=code`, required **scopes**, **client_id**, **state**, and a **URL-encoded** `redirect_uri` that matches your app registration.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** the token endpoint with `grant_type=authorization_code`, the code, redirect URI, and client credentials. Access tokens expire after **12 hours**. Request **`offline_access`** if you need a **refresh token** (refresh tokens expire after **365 days** without use).
  </Step>
  <Step title="Enable organization access">
    After token exchange, call **`GET`** `https://sandboxapi.deere.com/platform/organizations` (or production equivalent). If a link with **`rel=connections`** appears, redirect the user once per session to complete organization selection. Avoid redirect loops. See OAuth documentation for the full flow.
  </Step>
  <Step title="Call the Equipment API">
    Send **`Authorization: Bearer {token}`** on every request. Request only the scopes your app needs (for example **`eq1`** for read, **`eq2`** for create/update/delete equipment).
  </Step>
</Steps>

### Scopes used by this API

| Scope | Typical use |
| --- | --- |
| `eq1` | View equipment and reference data (**GET** list/detail and makes/types/models). |
| `eq2` | Create, update, or delete equipment contributed via this API (**POST**, **PUT**, **DELETE**). |
| `offline_access` | Refresh tokens (optional). |

## Paging

Collection responses use a **CollectionPage** shape: **`total`**, **`links`** (e.g. `nextPage` / `previousPage`), and **`values`**. Default page size is **10**. Use **`itemLimit`** (multiples of 10, up to **100**) and **`pageOffset`** to page through results.

## Endpoints

### Get equipment

Returns equipment for organizations the user has connected to your application. You can filter by organization, **machine** vs **implement**, serial numbers, **archived** state, and paging parameters. Requesting organizations the user cannot access can return **403 Forbidden** for the entire call.

```text
GET /isg/equipment
```

<ParamField query="organizationIds" type="string">
  Optional filter by organization id(s), as supported by the API.
</ParamField>

<ParamField query="category" type="string">
  Optional: `machine` or `implement`.
</ParamField>

<ParamField query="serialNumbers" type="string">
  Optional serial number filter.
</ParamField>

<ParamField query="archived" type="boolean">
  Optional filter for archived equipment.
</ParamField>

<ParamField query="itemLimit" type="integer">
  Page size: multiples of 10, max 100.
</ParamField>

<ParamField query="pageOffset" type="integer">
  Offset aligned with `itemLimit`.
</ParamField>

**OAuth scope:** `eq1`

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://equipmentapi.deere.com/isg/equipment?itemLimit=50&pageOffset=0" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/json' \
  --header 'Accept-Encoding: gzip'
```

</CodeGroup>

### Create equipment

Creates a **Machine** or **Implement** in the given organization. **201 Created** returns a **`Location`** header whose last segment is the new equipment id. Duplicate **`serialNumber`** in the same org returns **400 Bad Request**.

```text
POST /isg/organizations/{organizationId}/equipment
```

<ParamField path="organizationId" type="string" required>
  Operations Center organization id.
</ParamField>

**OAuth scope:** `eq2`

#### Body fields

<ParamField body="@type" type="string" required>
  `Machine` or `Implement`.
</ParamField>

<ParamField body="name" type="string" required>
  Display name in Operations Center (30 characters or fewer; unique within the org).
</ParamField>

<ParamField body="serialNumber" type="string">
  Optional; 30 characters or fewer; unique within the org when set.
</ParamField>

<ParamField body="model" type="array" required>
  Array containing at least one **`EquipmentModel`** object with **`id`** from the reference APIs.
</ParamField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url "https://equipmentapi.deere.com/isg/organizations/283480/equipment" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --data '{"@type":"Machine","name":"Equipment Name","serialNumber":"must_be_unique_string","model":[{"@type":"EquipmentModel","id":66280}]}'
```

</CodeGroup>

<Card title="Try it in the API Playground" icon="play" href="/POST /isg/organizations/{organizationId}/equipment">
  Interactive reference for **POST** `/isg/organizations/{organizationId}/equipment`.
</Card>

## Common error codes

| Code | Meaning |
| --- | --- |
| **400** | Invalid body, or invalid refresh token on OAuth token calls. |
| **401** | Missing license, expired token, or failed server authorization. |
| **403** | Access denied, missing scopes, restricted org, or invalid multi-org filter on **GET** `/isg/equipment`. |
| **404** | Resource id not found or no longer valid. |
| **429** | Rate limited; honor **`Retry-After`** when present. |
| **503** | Service unavailable; backoff or retry per **`Retry-After`**. |

For OAuth details, organization connections, and Postman setup, see the broader **OAuth 2** and **Precision Tech** documentation on Developer.Deere.com.
