## Overview

The **Operation Center Connection Management API** lets an application view its existing **Operations Center organization connections** and remove them when needed.

This API works only with an OAuth 2.0 **client credentials** token. The list endpoint also returns the **Operations Center permissions** assigned for each connection.

<CardGroup cols={2}>
  <Card title="List connections" icon="list">
    Retrieve all organization connections for the calling application.
  </Card>
  <Card title="Remove one connection" icon="unlink">
    Delete a single connection by `connectionId`.
  </Card>
  <Card title="Remove all partner connections" icon="trash">
    Delete all application connections for a specific organization.
  </Card>
  <Card title="Client credentials only" icon="lock">
    These endpoints do not support user tokens; use OAuth 2.0 `client_credentials`.
  </Card>
</CardGroup>

<Note>
  Permissions are assigned when a user connects an organization based on the OAuth scopes requested during user authorization and the highest permission level that user has in the organization. If a connection does not have the permissions you expect, request the right scopes during user authorization and have the user update the connection in `connections.deere.com`, or reconnect the organization.
</Note>

## Authentication

John Deere platform APIs use **OAuth 2.0**. This API supports only the **client credentials** grant type.

<Steps>
  <Step title="Create an application on Developer.Deere.com">
    When you create an application, it is assigned a **Client Key** and **Client Secret**. You sign API requests with these credentials.
  </Step>
  <Step title="Add a callback URL to your application">
    In the **Security** section of your application details, add one or more **Callback URLs (Redirect URIs)** and save your changes.

    Although this flow does not redirect a user, Deere recommends adding a placeholder such as `https://localhost:9090/callback` as part of the application registration process.
  </Step>
  <Step title="Call the OAuth 2 well-known URL">
    Retrieve the token endpoint from the Deere well-known metadata document:

    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Acquire an access token">
    Request a token using the `client_credentials` grant type. Put the client ID and secret in the HTTP `Authorization` header using **Basic Auth**, not in the request body.

    **Request URL:**

    ```bash
    POST https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token
    ```

    **Headers:**

    ```
    Authorization: Basic <base64(clientKey:clientSecret)>
    Accept: application/json
    Content-Type: application/x-www-form-urlencoded
    ```

    **Body:**

    ```
    grant_type=client_credentials&cache-control=no-cache&scope=<customScope>
    ```

    **Sample response:**

    ```json
    {
      "token_type": "Bearer",
      "expires_in": 43200,
      "access_token": "eyJhbG[...]1LQ",
      "scope": "customScope"
    }
    ```
  </Step>
  <Step title="Call the API with your access token">
    Send the token in the `Authorization` header of every request:

    ```bash
    Authorization: Bearer <access_token>
    ```
  </Step>
</Steps>

<Note>
  Deere notes that scopes are typically not required for a `client_credentials` token. If your integration uses a custom scope, request the value assigned to your application.
</Note>

## Endpoints

### Get list of connections

Retrieve all connections for the calling application based on the client in the token.

```text
GET /platform/connections
```

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://sandboxapi.deere.com/platform/connections" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/json'
```

</CodeGroup>

#### Response fields

<ResponseField name="connections" type="array">
  List of organization connections associated with the calling application.
</ResponseField>

<ResponseField name="connections[].connectionId" type="string">
  Unique identifier for the connection.
</ResponseField>

<ResponseField name="connections[].orgId" type="string">
  Operations Center organization identifier.
</ResponseField>

<ResponseField name="connections[].orgName" type="string">
  Display name of the connected organization.
</ResponseField>

<ResponseField name="connections[].permissions" type="array">
  Operations Center permissions currently assigned to the connection.
</ResponseField>

<ResponseField name="connections[].permissions[].scope" type="string">
  OAuth scope or permission family associated with the assigned permission.
</ResponseField>

<ResponseField name="connections[].permissions[].userPermission" type="string">
  Permission name assigned for the connection.
</ResponseField>

<ResponseField name="connections[].permissions[].code" type="string">
  Numeric or named permission code returned for the assigned permission.
</ResponseField>

<ResponseField name="connections[].permissions[].description" type="string">
  Human-readable description of the assigned permission.
</ResponseField>

### Delete connection by connection ID

Remove a specific connection by ID.

```text
DELETE /platform/connections/{connectionId}
```

<ParamField path="connectionId" type="string" required>
  Unique identifier of the connection to remove.
</ParamField>

<CodeGroup>

```bash cURL
curl --request DELETE \
  --url "https://sandboxapi.deere.com/platform/connections/2b76dfaf-6c2b-4ec1-baae-73d730db0b7c" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/json'
```

</CodeGroup>

Successful requests return `204 No Content`.

<Card title="Try it in the API Playground" icon="play" href="/DELETE /platform/connections/{connectionId}">
  Test this endpoint interactively with your client credentials access token.
</Card>

### Delete all partner connections by Org ID

Remove all connections between the calling application and the specified organization. This includes all partner connections.

```text
DELETE /platform/organizations/{orgId}/connections
```

<ParamField path="orgId" type="string" required>
  Operations Center organization identifier whose application connections should be removed.
</ParamField>

<CodeGroup>

```bash cURL
curl --request DELETE \
  --url "https://sandboxapi.deere.com/platform/organizations/1234567/connections" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/json'
```

</CodeGroup>

Successful requests return `204 No Content`.

<Card title="Try it in the API Playground" icon="play" href="/DELETE /platform/organizations/{orgId}/connections">
  Test this endpoint interactively with your client credentials access token.
</Card>
