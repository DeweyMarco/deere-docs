## Overview

A **Field Operation** represents data collected while working in a field—primarily tillage, seeding, application, and harvest layers. The Field Operations APIs expose much of the same information available from **Field Analyzer** in MyJohnDeere Operations Center.

Using the Field Operations APIs, you can access:

- Metadata about each Field Operation.
- Totals for area worked and total amount planted, applied, or harvested.
- Map images (with legends) showing what happened in each part of the field.
- Point-level data via **shapefile** export.

<Note>
  New operational data can appear quickly, and Field Operations can be large. If you plan to poll frequently, consider the **Data Subscription Service** for webhooks when new Field Operations are available.
</Note>

### Field Operations vs Map Layers

You can access map images through **Field Operations** or the **Map Layers** API, but they represent different data. Field Operations are processed internally by **Field Analyzer**. **Map Layers** are data contributed into Operations Center by another tool.

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scopes (**`ag2`** for Field Operations), **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**. Request **`offline_access`** for a **refresh token**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call Field Operations">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`** (unless requesting a map image media type).
  </Step>
</Steps>

### OAuth scope for these endpoints

| Scope | Description |
| --- | --- |
| `ag2` | Locations Access Level 2 — view locations and **analyze production data** (required for all Field Operations endpoints in this guide) |

Request only scopes your application needs. User permissions cap what the connection can do.

## Paging and links

List responses may use **CollectionPage** with **`total`**, **`links`** (`nextPage` / `previousPage`), and **`values`**. Default page size is often **10**. Use **`itemLimit`** (multiples of **10**, max **100**) and **`pageOffset`** where supported.

Use **`showLinks=none`** or filtered **`showLinks`** values to reduce HATEOAS payload size.

## Shapefiles overview

A **shapefile** is an industry-standard format for geospatial data. MyJohnDeere exports use **MultiPoint** or **MultiPolygon** shapes. Use standard GIS libraries to read shapefiles. Example shapefiles for harvest, seeding, and application are published on GitHub in Deere’s samples.

Exported shapefiles include:

| File | Purpose |
| --- | --- |
| `.dbf` | Data attributes per shape (dBase IV). Columns depend on operation type. |
| `.prj` | Projection — exports use **WGS84** (wkid **4326**). |
| `.shp` | Feature geometry (points or polygons). |
| `.shx` | Index for geometry. |
| `.json` | Extra metadata (not ESRI standard), including **units of measure** per DBF column. |

Units in documentation may list metric and imperial; the actual system is determined by the **`Accept-UOM-System`** header on the shapefile request when available, otherwise by organization preferences. The bundled `.json` lists exact units.

### Shape type and resolution

Optional query parameters **`shapeType`** and **`resolution`** strongly affect file size. Provide both when possible.

| ShapeType | Resolution | Description | Approx. size |
| --- | --- | --- | --- |
| Point | EachSection | Default for backwards compatibility; very large duplicate data. | Up to ~65 GB |
| Point | EachSensor | One point per sensor reading, centered in the sensor area. | Up to ~250 MB |
| Point | OneHertz | ~One sensor reading per second. | Up to ~75 MB |
| Polygon | EachSection | Not supported. | — |
| Polygon | EachSensor | Polygon per sensor reading (implement path). | Up to ~350 MB |
| Polygon | OneHertz | Polygon per reading, ~1 Hz. | Up to ~120 MB |

## Field Operation events (Data Subscription Service)

Field operation events fire on add, update, or delete. Subscriptions require an **`orgId`** filter; other filters (`cropSeason`, `fieldId`, `fieldOperationType`) are optional. Prefer **one org per subscription** because org lists cannot change after creation and subscriptions end if the user loses access to any listed org.

A **404** when following a **`targetResource`** link from an event can mean the Field Operation was deleted.

### Asynchronous shapefile download

```text
GET /platform/fieldOps/{operationId}
```

<ParamField path="operationId" type="string" required>
  Field operation id.
</ParamField>

<ParamField query="shapeType" type="string">
  `Point` or `Polygon`.
</ParamField>

<ParamField query="resolution" type="string">
  `EachSection`, `EachSensor`, or `OneHertz` (see shape type / resolution table above).
</ParamField>

<ParamField header="Accept-UOM-System" type="string">
  Influences metric/imperial in export when supported.
</ParamField>

**Responses:** **202** (poll), **307** (follow **`Location`** pre-signed URL without OAuth), **406** (cannot generate). Poll with backoff from **5** seconds: `5 * 2^(attempt - 1)`.

**OAuth scope:** `ag2`

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://sandboxapi.deere.com/platform/fieldOps/a1b2c3d4-e5f6-7890-abcd-ef1234567890?shapeType=Point&resolution=EachSensor" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.axiom.v3+json' \
  --header 'Accept-UOM-System: metric' \
  --dump-header -
```

</CodeGroup>

## Additional notes

- **ADAPT-style units:** Values like `gal1ac-1` follow ADAPT conventions (powers on numerator/denominator).
- **Response evolution:** Application and tank-mix operations may expose `products` and `applicationProductTotals` in measurements; migrate off legacy single `product` / `productTotals` fields per platform communications.
