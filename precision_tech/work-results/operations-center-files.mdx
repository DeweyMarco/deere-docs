## Overview

Nearly every file exposed through the MyJohnDeere **Files** APIs is a **compressed zip** archive. Zip contents vary by format; Operations Center supports many proprietary and standard types. For proprietary Deere formats, **ADAPT** plugins can read and write data (see Deere offline SDK documentation).

Using the Files APIs, you can:

- **List** files globally or per organization, with HATEOAS links to ownership and sharing.
- **Create** a file id, then **upload** zip/binary content in a second request.
- **Download** or inspect metadata; use **chunked** download for files **50 MB or larger**.
- **Transfer** files to WDT-capable displays (subject to an **800 MB** transfer limit).

<CardGroup cols={2}>
  <Card title="Two-step upload" icon="upload">
    **POST** `/platform/organizations/{orgId}/files` creates the record ( **`Location`** header has the new id), then **PUT** `/platform/files/{fileId}` sends the bytes.
  </Card>
  <Card title="Chunked download" icon="download">
    Full download works under **50 MB**. Above that, use **`Range`** or **`offset`** / **`size`** (Range wins if both are sent).
  </Card>
  <Card title="File transfers (WDT)" icon="wifi">
    **POST** `/platform/organizations/{orgId}/fileTransfers` queues a file to a machine; poll **GET** `/platform/fileTransfers` or **GET** `/platform/fileTransfers/{id}` for status.
  </Card>
  <Card title="OAuth 2.0 (`files` + `ag3`)" icon="lock">
    File CRUD endpoints require **`files`** and **`ag3`**. File transfer endpoints require **`files`** only. Complete **connections** when **GET** `/platform/organizations` returns a **`connections`** link.
  </Card>
</CardGroup>

<Note>
  For operational **documentation** data without managing zip files directly, consider the **Field Operations** API as a simpler consumption path.
</Note>

### File types (conceptual)

Files are classified by the **type of data** they contain. One zip can include multiple data types; labeling follows priority (for example, setup + Rx may be labeled **Setup**; setup + documentation may be labeled **Doc**).

| Category | Description |
| --- | --- |
| **Setup** | Configuration to prepare a machine for a task: fields, boundaries, guidance, products, jobs/tasks/work orders, etc. |
| **Rx** | Prescription maps with zones and application rates (seeding or application). |
| **Doc** | Data recorded in the field (sensors, point-by-point location). |

## Rx shapefiles (ESRI)

John Deere displays support proprietary prescriptions and **ESRI shapefile**-based prescriptions. A shapefile prescription requires:

- The shapefile must live in a directory named **`Rx`**.
- A complete shapefile includes **`.shp`**, **`.shx`**, and **`.dbf`**.
- Geometries must be **Polygon** or **MultiPolygon** using **WGS84** coordinates.
- At least one numeric **Rate** column (unit optional; user may confirm units at selection). Optional **`zone_color`** column (ARGB).

**Suggested zip naming**

```text
<Client>_<Farm>_<Field>_<Product>.zip
Rx/
  <Client>_<Farm>_<Field>_<Product>.shp
  <Client>_<Farm>_<Field>_<Product>.shx
  <Client>_<Farm>_<Field>_<Product>.dbf
```

**Rate column naming (≤ 10 characters)** helps users pick units on displays, for example: `gal_p_ac`, `floz_p_ac`, `ml_p_ha`, `L_p_ha`, `lb_p_ac`, `g_p_ac`, `kg_p_ac`, `ton_p_ac`, `tonnes_p_h`.

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scopes (**`files`**, **`ag3`** for file CRUD; **`files`** for transfers), **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**. Request **`offline_access`** for a **refresh token**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call Files APIs">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`** for JSON responses (unless downloading binary content).
  </Step>
</Steps>

### OAuth scopes for these endpoints

| Scope | Typical use |
| --- | --- |
| `files` | Files API access and file transfers (required for all endpoints in this guide). |
| `ag3` | Required **with** `files` for list/create/upload/download file endpoints (manage locations and production data). |

Request only scopes your application needs. User permissions cap what the connection can do.

## Request headers (typical)

| Header | Value |
| --- | --- |
| `Authorization` | `Bearer {access_token}` |
| `Accept` | `application/vnd.deere.axiom.v3+json` (metadata and collections) |
| `Content-Type` | `application/vnd.deere.axiom.v3+json` (JSON bodies); `application/zip` or `application/octet-stream` (uploads) |
| `Range` | `bytes=start-end` (optional chunked download) |
| `Accept-Encoding` | `gzip` (optional; where supported) |

## Paging and links

List responses may use **CollectionPage** with **`total`**, **`links`** (`nextPage` / `previousPage`), and **`values`**. Default page size is often **10**. Use **`itemLimit`** (multiples of **10**, max **100**) and **`pageOffset`** where supported.

Use **`showLinks=none`** or filtered **`showLinks`** values to reduce HATEOAS payload size.

### View or download a file

```text
GET /platform/files/{fileId}
```

<ParamField path="fileId" type="string" required>
  File id from **POST** create response or lists.
</ParamField>

<ParamField query="offset" type="integer">
  Byte offset for chunked download (use with **`size`**).
</ParamField>

<ParamField query="size" type="integer">
  Byte length for chunked download.
</ParamField>

<ParamField header="Range" type="string">
  `bytes=start-end`; overrides **`offset`** / **`size`** if both are provided.
</ParamField>

<ParamField header="Accept" type="string">
  `application/vnd.deere.axiom.v3+json` for metadata; `application/zip`, `application/octet-stream`, or related types for binary download.
</ParamField>

**OAuth scopes:** `files`, `ag3`

<CodeGroup>

```bash cURL (metadata)
curl --request GET \
  --url "https://sandboxapi.deere.com/platform/files/55" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.axiom.v3+json'
```

```bash cURL (download)
curl --request GET \
  --url "https://sandboxapi.deere.com/platform/files/55" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/zip' \
  --output file.zip
```

</CodeGroup>

### Upload or update a file

```text
PUT /platform/files/{fileId}
```

<ParamField path="fileId" type="string" required>
  Target file id (created with **POST** `/platform/organizations/{orgId}/files`).
</ParamField>

**Content-Type** may be `application/octet-stream`, `application/zip`, `application/x-zip`, `application/x-zip-compressed`, `multipart/form-data`, or `multipart/mixed`.

**OAuth scopes:** `files`, `ag3`

<CodeGroup>

```bash cURL
curl --request PUT \
  --url "https://sandboxapi.deere.com/platform/files/55" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.axiom.v3+json' \
  --header 'Content-Type: application/zip' \
  --data-binary @upload.zip
```

</CodeGroup>

## Additional notes

- **HATEOAS:** Responses include **links** for related resources (`owningOrganization`, `partnerships`, `initiateFileTransfer`, `wdtCapableMachines`, etc.). Follow only links your token can access.
- **Compression:** Many APIs honor **`Accept-Encoding: gzip`** for responses.
- **Errors:** Handle **400**, **401**, **403** (including missing scopes and RCA flows), **404**, **429** (respect **`Retry-After`**), and **503** per platform guidance.
