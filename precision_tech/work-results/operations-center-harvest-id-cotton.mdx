## Overview

**Harvest ID (HID) Cotton** ties cotton harvest information to each **individual cotton module**. Data is logged per module, and an **RFID** chip in the module wrap lets farmers and gins track modules from the field through the gin to final grading.

With the HID Cotton API you can access attributes such as:

- Organization and field identifiers
- Module **serial number** and **RFID** number
- **Wrap** date, time, and location
- **Drop** location
- **Variety**, **moisture**, **diameter**, **weight**, **area**
- **Machine PIN**, **operator**
- **Gin** name, **producer** name
- **Comments**

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scopes (**`files`** and **`ag2`** for HID Cotton), **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**. Request **`offline_access`** for a **refresh token**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call HID Cotton APIs">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`**.
  </Step>
</Steps>

### OAuth scopes for these endpoints

| Scope | Description |
| --- | --- |
| `files` | Files API access (required together with `ag2` for HID Cotton). |
| `ag2` | Locations Access Level 2 — analyze production data (required together with `files`). |

Request only scopes your application needs. User permissions cap what the connection can do.

## Paging

List responses may use **CollectionPage** with **`total`**, **`links`**, and **`values`**. Use **`itemLimit`** (multiples of **10**, up to **2000**) and **`pageOffset`** to walk pages. Optional **`showLinks`** can reduce link payload size (for example `none`).

---

## Error handling (summary)

| HTTP | Meaning |
| --- | --- |
| **400** | Bad request or invalid refresh token on token calls. |
| **401** | Expired or invalid token; refresh or re-authenticate. |
| **403** | Access denied — wrong resource, missing scopes, missing org connection, or **RCA** (check **`X-Deere-Warning`**, **`X-Deere-Terms-Location`**). |
| **404** | Resource id not found or no longer valid for the user. |
| **429** | Rate limited — honor **`Retry-After`** when present. |
| **503** | Service unavailable — backoff or **`Retry-After`**. |

## Endpoints

### List HID Cotton modules

```text
GET /platform/organizations/{orgId}/harvestIdentificationModules
```

<ParamField path="orgId" type="string" required>
  Organization id.
</ParamField>

<ParamField query="itemLimit" type="integer">
  Page size: multiples of 10, max 2000.
</ParamField>

<ParamField query="pageOffset" type="integer">
  Pagination offset (aligned with `itemLimit`).
</ParamField>

<ParamField query="showLinks" type="string">
  Omit or filter links (`none` or comma-separated `rel` names).
</ParamField>

**OAuth scopes:** `files`, `ag2`

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://sandboxapi.deere.com/platform/organizations/283480/harvestIdentificationModules?itemLimit=50&pageOffset=0" \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.axiom.v3+json'
```

</CodeGroup>
