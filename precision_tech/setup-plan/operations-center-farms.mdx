## Overview

A **farm** is a set of **fields** associated with a **client**. Use these endpoints to manage farms under an organization and to follow **`links`** to related **fields**, **clients**, and the **owning organization**.

<CardGroup cols={2}>
  <Card title="List & read" icon="list">
    **GET** `/platform/organizations/{orgId}/farms` and **GET** `/platform/organizations/{orgId}/farms/{farmId}` — discover farms; single-farm responses link to **`fields`**, **`clients`**, and **`owningOrganization`**.
  </Card>
  <Card title="Create & update" icon="pencil">
    **POST**, **PUT**, and **DELETE** require **Locations Level 3** (**`ag3`**).
  </Card>
  <Card title="Clients & fields" icon="map">
    **GET** `/platform/organizations/{orgId}/farms/{farmId}/clients` and **GET** `.../fields` — related location resources (**`ag1`**).
  </Card>
  <Card title="Events & OAuth" icon="bell">
    **Farm events** (create, delete, modify) are delivered via the **Data Subscription Service**; subscriptions require an **`orgId`** filter. Complete **connections** when **`connections`** appears on **GET** `/platform/organizations`.
  </Card>
</CardGroup>

## Farm events (Data Subscription Service)

**Farm events** fire when a farm is **created**, **modified**, or **deleted** (for example when metadata changes, a farm is archived or unarchived, or new farm data is available). Subscriptions are created through the **Data Subscription Service**; the caller must supply an **`orgId`** filter. Other filters are optional. Filter values are **arrays**, so multiple **`orgId`** or **`farmId`** values can be included in one subscription.

Prefer **one organization per subscription**: you cannot change the list of **`orgId`** values after creation, and the subscription ends if the user loses access to any listed organization.

### Farm event statuses

| Status | Description |
| --- | --- |
| **ARCHIVED** | Farm is archived. |
| **ACTIVE** | Farm is not archived and is available to use. |

### Farm event actions

| Action | Description |
| --- | --- |
| **CREATED** | Farm was created. |
| **DELETED** | Farm was deleted. |
| **MODIFIED** | Metadata update, archive, or unarchive. |

### Farm event filters (metadata)

| Filter | Required | Example | Description |
| --- | --- | --- | --- |
| **orgId** | Yes | `"123456"` | Organization the farm belongs to. |
| **farmId** | No | UUID | Specific farm. |
| **status** | No | `"AVAILABLE"` | Farm status. |
| **action** | No | `"CREATED"` | Action that occurred. |

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scopes (**`ag1`** for read, **`ag3`** for create/update/delete farms), **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**. Request **`offline_access`** for a **refresh token**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call Farms APIs">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`**. For **POST**/**PUT**, send **`Content-Type: application/vnd.deere.axiom.v3+json`**.
  </Step>
</Steps>

### OAuth scopes for these endpoints

| Scope | Description |
| --- | --- |
| `ag1` | View locations — **GET** list, **GET** farm, **GET** clients, **GET** fields |
| `ag3` | Manage locations — **POST**, **PUT**, **DELETE** farms |

Request only scopes your application needs. User permissions cap what the connection allows.

## Paging and links

List responses may use **CollectionPage** with **`total`**, **`links`**, and **`values`**. Use **`itemLimit`** (multiples of **10**, max **100**) and **`pageOffset`** where supported. Use **`showLinks=none`** or filtered **`showLinks`** to trim HATEOAS payloads.

Example **`links`** on a farm may include **`self`**, **`fields`**, **`clients`**, and **`owningOrganization`**.

## Additional notes

- **403 / terms:** Some orgs return **403** with **X-Deere-Terms-Location** when user action is required (RCA). Navigate the user to that URL when possible.
- **Discovery:** Follow **`links`** from **GET** `/platform/organizations` and farm responses rather than hard-coding resource URLs.
