## Overview

**Products** are the digital representation of physical inputs—**varieties**, **chemicals**, **fertilizers**, **tank mixes**, and **dry blends**—used when planning work, executing plans, and analyzing results in **Operations Center**. These APIs let you **read** an organization’s lists, **add** custom items, **update** them, **search** reference catalogs, and **associate** reference products to an organization.

<CardGroup cols={2}>
  <Card title="Varieties" icon="leaf">
    List and maintain **seed varieties**; search the **reference** catalog; set **overrides** when importing reference varieties.
  </Card>
  <Card title="Chemicals & fertilizers" icon="flask">
    CRUD for org-scoped **chemicals** and **fertilizers**, plus **reference** search, **documents**, and **PATCH** overrides.
  </Card>
  <Card title="Mixes & blends" icon="layers">
    **Tank mixes** and **dry blends** combine components that must already exist on the organization.
  </Card>
  <Card title="Supporting resources" icon="file-text">
    **Product companies**, **active ingredients** (`/isg`), and **document** payloads with embedded PDFs.
  </Card>
</CardGroup>

### Products catalog (categories)

| Category | Description | Example |
| --- | --- | --- |
| Crop types | Commodity for the year; varieties must use a supported crop type from **`/cropTypes`**. | `CORN_WET` (shown as “Corn” in UIs) |
| Varieties | Branded seed for a crop type | Pioneer variety for corn |
| Chemicals | Crop protection (herbicide, insecticide, fungicide, etc.) | Glyphosate products |
| Fertilizers | Nutrients and manure | Dry fertilizer analysis |
| Tank mixes | One or more chemicals/fertilizers plus a **carrier** (e.g. water) | Spray tank recipe |
| Dry blends | Two or more chemicals or fertilizers | Dry blend product |

### Where products appear

Customers manage products in **Product Manager** in Operations Center. Products are also used in **Setup File Creator**, **Work Planner**, **Field Analyzer**, and related tools.

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** (authorization code grant). Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scopes **`eq1`** and/or **`eq2`** (and **`offline_access`** if you need refresh tokens), **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call Products APIs">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`**. For **POST**, **PUT**, and **PATCH**, also send **`Content-Type: application/vnd.deere.axiom.v3+json`**.
  </Step>
</Steps>

## Getting started

1. **Read existing lists** — **GET** organization **varieties**, **chemicals**, **fertilizers**, and **tank mixes** so you do not duplicate entries.
2. **Resolve crop types** — Before **POST** varieties, use **GET** `/platform/cropTypes` (or org crop types) so **`cropName`** is valid.
3. **Create** — **POST** custom varieties, chemicals, fertilizers, tank mixes, or dry blends; or **POST** `associateToOrg` after finding a reference **erid**.
4. **Update** — **PUT** or **PATCH** (overrides) when correcting metadata, archive state, or composition.

<Note>
  Reference **POST** bodies may be empty where the platform allows; custom **POST**/**PUT** payloads must match the live Axiom contract for each resource type.
</Note>

---

## Paging, links, and search

- List responses may use **CollectionPage** with **`itemLimit`** (multiples of **10**, max **100**) and **`pageOffset`**.
- **`GET /platform/varieties`**, **`GET /platform/chemicals`**, and **`GET /platform/fertilizers`** require **one of** `searchString`, `productName`, or `brandName`. Results are capped at **100** items.

## Additional type information

### Chemicals — material classification and type

**Material classification:** `LIQUID`, `DRY`, `GAS`.

**Type:** `ADDITIVE`, `FUNGICIDE`, `INSECTICIDE`, `HERBICIDE`, `GROWTH_REGULATOR`, `NITROGEN_STABILIZER`, `ADJUVANT`, `DEFOLIANT`.

### Fertilizers — material classification and type

**Material classification:** `LIQUID`, `DRY`, `GAS`.

**Type:** `FERTILIZER`, `MANURE`.

### Tank mixes — units

Material classification values align with chemicals/fertilizers. Rate **units** may use **ADAPT**-style strings (for example `gal1ac-1`, `floz1ac-2`). See John Deere **ADAPT** unit documentation for the full set of allowed unit strings.
