## Overview

**Crop types** identify crops used in **Operations Center** and related precision workflows. These endpoints return the **system catalog**, a **single** crop type by **name** or **id** (same URL pattern), or crop types **for a specific organization**.

<CardGroup cols={2}>
  <Card title="Global catalog" icon="list">
    **GET** `/platform/cropTypes` lists available crop types. **GET** `/platform/cropTypes/{name}` returns one crop type; the path value may be a **name** or **id** per platform behavior.
  </Card>
  <Card title="Organization scope" icon="building">
    **GET** `/platform/organizations/{organizationId}/cropTypes` lists crop types for that organization.
  </Card>
  <Card title="Localization" icon="globe">
    Optional **Accept-Language** selects preferred language for localized strings. See [Accept-Language](#accept-language) for supported tags.
  </Card>
  <Card title="OAuth" icon="key">
    All endpoints require **`ag1`**. Complete **connections** when **`connections`** appears on **GET** `/platform/organizations`.
  </Card>
</CardGroup>

### Endpoints overview

| Operation | Method & path |
| --- | --- |
| Retrieve all crop types | **GET** `/platform/cropTypes` |
| View a specific crop type | **GET** `/platform/cropTypes/{name}` |
| Retrieve crop types for an organization | **GET** `/platform/organizations/{organizationId}/cropTypes` |

<Note>
  The platform documents **GET** `/cropTypes/{name}` and **GET** `/cropTypes/{id}` as the same route: one path segment identifies the crop type by **name** or **id**. This reference uses **`{name}`** in the path; pass whichever identifier your workflow uses.
</Note>

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scope **`ag1`**, **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**. Request **`offline_access`** for a **refresh token**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call Crop Types APIs">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`**.
  </Step>
</Steps>

## Paging and links

List responses may use **CollectionPage** with **`total`**, **`links`**, and **`values`**. Where supported, use **`itemLimit`** (multiples of **10**, max **100**) and **`pageOffset`**. Use **`showLinks=none`** or filtered **`showLinks`** to trim HATEOAS payloads.

## Additional notes

- **403 / terms:** Some orgs return **403** with **X-Deere-Terms-Location** when user action is required (RCA). Navigate the user to that URL when possible.
- **Discovery:** Follow **`links`** from **GET** `/platform/organizations` rather than hard-coding resource URLs.
