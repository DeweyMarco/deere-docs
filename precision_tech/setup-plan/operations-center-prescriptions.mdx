## Overview

A **prescription (Rx)** is a set of agronomic instructions for **application**, **seeding**, and **tillage** operations. It applies inputs—such as seeds, fertilizers, pesticides, or tillage depth—at **variable rates** across **zones** within a field.

Prescriptions are built to **optimize productivity**, **reduce input costs**, and support **sustainable** farming practices.

<CardGroup cols={2}>
  <Card title="Variable-rate zones" icon="map">

  </Card>
  <Card title="Operations" icon="tractor">

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**, and request the scopes required for **Files**, **Map Layers**, and other resources you call.

<Steps>
  <Step title="Register your application">
    In the developer portal, register **Redirect URIs** that match your app exactly; mismatched URIs can return **400 Bad Redirect**.
  </Step>
  <Step title="Request tokens">
    Complete the authorization code flow to obtain **access** and **refresh** tokens. Include an `Authorization` header whose value is `Bearer` followed by your access token.
  </Step>
  <Step title="Use the right base URL">
    Use the documented **sandbox** or **production** host for each API (for example `https://sandboxapi.deere.com` for platform APIs). Paths belong in the request URL, not in the server host.
  </Step>
</Steps>

For step-by-step examples, see any **Operations Center** API guide in this section (for example [Fields](/precision_tech/setup-plan/operations-center-fields))—the OAuth pattern is the same across Precision Tech APIs.
