## Overview

**Flags** mark locations or observations on **fields**; **flag categories** classify flags (reference, master, or custom). Use these endpoints to **list**, **create**, **update**, and **delete** flags and categories, and to read or update **flag category preferences** (for example default **hexColor**).

<CardGroup cols={2}>
  <Card title="Flags" icon="flag">
    **GET** list and single-flag reads require **`ag1`**. **POST**, **PUT**, and **DELETE** on flags require **`ag3`**. List flags for a field with **GET** `.../fields/{fieldId}/flags`.
  </Card>
  <Card title="Categories & preferences" icon="tags">
    **Flag categories** follow the same scope pattern (**`ag1`** read, **`ag3`** write). **FlagCategoryPreference** includes a **`default`** object ( **`prefKey`:** `default`, **`hexColor`:** `#FFFFFF` ) created on first access.
  </Card>
  <Card title="Events & OAuth" icon="bell">
    **Flag** and **Flag Category** events use the **Data Subscription Service**; subscriptions require an **`orgId`** filter. Complete **connections** when **`connections`** appears on **GET** `/platform/organizations`.
  </Card>
  <Card title="Paging & links" icon="link">
    Lists may use **CollectionPage** with **`itemLimit`** / **`pageOffset`**. Trim HATEOAS with **`showLinks`**.
  </Card>
</CardGroup>

## Flag events (Data Subscription Service)

**Flag events** fire when a flag is **added**, **modified**, or **deleted** (for example new flag data, edits, archive/unarchive, or field merge/unmerge/archive behavior per platform). Subscriptions are created through the **Data Subscription Service**; the caller must supply an **`orgId`** filter. Other filters are optional. Filter values are **arrays**, so multiple **`orgId`**, **`flagId`**, or other values may appear in one subscription.

Prefer **one organization per subscription**: you cannot change the list of **`orgId`** values after creation, and the subscription ends if the user loses access to any listed organization.

### Flag event statuses

| Status | Description |
| --- | --- |
| **ARCHIVED** | Flag is archived. |
| **AVAILABLE** | Flag is not archived and is available to use. |

### Flag event actions

| Action | Description |
| --- | --- |
| **CREATED** | Flag was created. |
| **DELETED** | Flag was deleted. |
| **MODIFIED** | Metadata update, archive, unarchive, field merge/unmerge, field archive/unarchive. |

### Flag event filters (metadata)

| Filter | Required | Example | Description |
| --- | --- | --- | --- |
| **orgId** | Yes | `"123456"` | Organization the flag belongs to. |
| **flagId** | No | UUID | The flag itself. |
| **status** | No | `"AVAILABLE"` | Flag status. |
| **action** | No | `"CREATED"` | Action that occurred. |

## Flag category events (Data Subscription Service)

**Flag category events** fire when a category is **added**, **modified**, or **deleted**. The caller must supply an **`orgId`** filter; other filters are optional. Filter values are **arrays**.

Prefer **one organization per subscription** for the same reasons as flag events.

### Flag category event statuses

| Status | Description |
| --- | --- |
| **ARCHIVED** | Flag category is archived. |
| **AVAILABLE** | Flag category is not archived and is available to use. |

### Flag category event actions

| Action | Description |
| --- | --- |
| **CREATED** | Flag category was created. |
| **DELETED** | Flag category was deleted. |
| **MODIFIED** | Metadata update, archive, or unarchive. |

### Flag category event filters (metadata)

| Filter | Required | Example | Description |
| --- | --- | --- | --- |
| **orgId** | Yes | `"123456"` | Organization the flag category belongs to. |
| **flagCategoryId** | No | UUID | The flag category itself. |
| **status** | No | `"AVAILABLE"` | Flag category status. |
| **action** | No | `"CREATED"` | Action that occurred. |

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scopes (**`ag1`** for read, **`ag3`** for create/update/delete flags and categories), **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**. Request **`offline_access`** for a **refresh token**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call Flags APIs">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`**. For **POST**/**PUT**, send **`Content-Type: application/vnd.deere.axiom.v3+json`**.
  </Step>
</Steps>

### OAuth scopes for these endpoints

| Scope | Description |
| --- | --- |
| `ag1` | View locations — **GET** lists and single-resource reads for flags, categories, and preferences |
| `ag3` | Manage locations — **POST**, **PUT**, **DELETE** flags and categories; **PUT** preferences |

Request only scopes your application needs. User permissions cap what the connection allows.

- **403 / terms:** Some orgs return **403** with **X-Deere-Terms-Location** when user action is required (RCA). Navigate the user to that URL when possible.
- **Discovery:** Follow **`links`** from organization and flag responses rather than hard-coding resource URLs.
