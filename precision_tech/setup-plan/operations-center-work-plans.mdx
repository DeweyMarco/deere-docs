## Overview

**Work Plan** integration automates setup and reduces operator guesswork in the field. Use it to help customers:

1. **Plan work from anywhere**, with any software.
2. **Prepare work** to be transferred to equipment via **Work Planner** in Operations Center.
3. **Set up Generation 4 displays** (4200, 4240, 4600, 4640) and **Generation 5 displays (G5)** with one-touch workflows.
4. **Analyze completed work** in Operations Center or from another software solution.

## Where do Work Plans appear?

| Surface | Role |
| --- | --- |
| **Work Planner (Operations Center web and mobile)** | Customers view and finalize plans, add setup such as guidance and equipment, and send plans to machines. |
| **Generation 4 displays** (4200, 4240, 4600, 4640) | Work Plans can be sent wirelessly for in-cab execution. |
| **Generation 5 displays (G5)** | Work Plans can be sent wirelessly for in-cab execution. |

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**, and request the scopes required for each resource you call (for example locations, files, and work-plan–related operations per the developer portal).

<Steps>
  <Step title="Register your application">
    Register **Redirect URIs** that match your app exactly; mismatched URIs can return **400 Bad Redirect**.
  </Step>
  <Step title="Request tokens">
    Complete the authorization code flow to obtain **access** and **refresh** tokens. Send **`Authorization: Bearer {access_token}`** on API requests.
  </Step>
  <Step title="Use the correct host and paths">
    Use the documented **sandbox** or **production** base URL for each API. Keep the server URL as the **bare host** (for example `https://sandboxapi.deere.com` for many platform APIs)—put path segments in the request path, not in the host.
  </Step>
</Steps>

For detailed OAuth steps and header examples, see any **Operations Center** API guide in this section (for example [Fields](/precision_tech/setup-plan/operations-center-fields))—the pattern is the same across Precision Tech APIs.
