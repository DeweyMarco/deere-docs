## Overview

A **client** is the **grower** and/or **Operations Center user** tied to a specific **farm** in the location hierarchy. Use these endpoints to manage clients under an organization and to follow **`links`** to related **fields**, **farms**, and the **owning organization**.

<CardGroup cols={2}>
  <Card title="List & read" icon="list">
    **GET** `/platform/organizations/{orgId}/clients` and **GET** `/platform/organizations/{orgId}/clients/{clientId}` — discover clients; single-client responses link to **`fields`**, **`farms`**, and **`owningOrganization`**.
  </Card>
  <Card title="Create & update" icon="pencil">
    **POST**, **PUT**, and **DELETE** require **Locations Level 3** (**`ag3`**). New clients are created with **active** status.
  </Card>
  <Card title="Farms & fields" icon="map">
    **GET** `/platform/organizations/{orgId}/clients/{clientId}/farms` and **GET** `.../fields` — related location resources; **`fields`** requires **`ag1`**.
  </Card>
  <Card title="Events & OAuth" icon="bell">
    **Client events** (create, delete, modify) are delivered via the **Data Subscription Service**; subscriptions require an **`orgId`** filter. Complete **connections** when **`connections`** appears on **GET** `/platform/organizations`.
  </Card>
</CardGroup>

## Client events (Data Subscription Service)

**Client events** fire when a client is **created**, **modified**, or **deleted** (for example when metadata changes or a client is archived or unarchived). Subscriptions are created through the **Data Subscription Service**; the caller must supply an **`orgId`** filter. Other filters are optional. Filter values are **arrays**, so multiple **`orgId`** or **`clientId`** values can be included in one subscription.

Prefer **one organization per subscription**: you cannot change the list of **`orgId`** values after creation, and the subscription ends if the user loses access to any listed organization.

### Client event statuses

| Status | Description |
| --- | --- |
| **ARCHIVED** | Client is archived. |
| **ACTIVE** | Client is not archived and is available to use. |

### Client event actions

| Action | Description |
| --- | --- |
| **CREATED** | Client was created. |
| **DELETED** | Client was deleted. |
| **MODIFIED** | Metadata update, archive, or unarchive. |

### Client event filters (metadata)

| Filter | Required | Example | Description |
| --- | --- | --- | --- |
| **orgId** | Yes | `"123456"` | Organization the client belongs to. |
| **clientId** | No | UUID | Specific client. |
| **status** | No | `"AVAILABLE"` | Client status. |
| **action** | No | `"CREATED"` | Action that occurred. |

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scopes (**`ag1`** for read, **`ag3`** for create/update/delete clients), **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**. Request **`offline_access`** for a **refresh token**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call Clients APIs">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`**. For **POST**/**PUT**, send **`Content-Type: application/vnd.deere.axiom.v3+json`**.
  </Step>
</Steps>

### OAuth scopes for these endpoints

| Scope | Description |
| --- | --- |
| `ag1` | View locations — **GET** list, **GET** client, **GET** farms, **GET** fields |
| `ag3` | Manage locations — **POST**, **PUT**, **DELETE** clients |

Request only scopes your application needs. User permissions cap what the connection allows.

## Paging and links

List responses may use **CollectionPage** with **`total`**, **`links`**, and **`values`**. Use **`itemLimit`** (multiples of **10**, max **100**) and **`pageOffset`** where supported. Use **`showLinks=none`** or filtered **`showLinks`** to trim HATEOAS payloads.

## Additional notes

- **403 / terms:** Some orgs return **403** with **X-Deere-Terms-Location** when user action is required (RCA). Navigate the user to that URL when possible.
- **Discovery:** Follow **`links`** from **GET** `/platform/organizations` and client responses rather than hard-coding resource URLs.
