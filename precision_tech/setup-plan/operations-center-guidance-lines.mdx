## Overview

**Guidance lines** are polymorphic resources attached to a **field**. They describe steering geometry such as **AB lines**, **AB curves**, **adaptive curves**, and **circle tracks**. Use **`@type`** on each object to determine the concrete subtype.

<CardGroup cols={2}>
  <Card title="List & read" icon="list">
    **GET** collection and **GET** by id return guidance line objects. By default, the list returns **active** lines only. This API **does not** support **eTags** on the list endpoint.
  </Card>
  <Card title="Create & update" icon="pencil">
    **POST** currently supports **AB Line** creation only (**`ag3`**). **PUT** updates the guidance line **name** (**`ag3`**).
  </Card>
  <Card title="Geometry & shapes" icon="map">
    Curves may include a **shapes** array of **MultiPoint** sections when shapes are embedded. See **dtProjectionType**, **dtABLineSaveMethod**, and related representation-system values in John Deere documentation.
  </Card>
  <Card title="Events & OAuth" icon="bell">
    **GuidanceLine** events are delivered via the **Data Subscription Service**; subscriptions require an **`orgId`** filter. Complete **connections** when **`connections`** appears on **GET** `/platform/organizations`.
  </Card>
</CardGroup>

### Endpoints overview

## GuidanceLine object (subtypes)

The guidance line API is **polymorphic**. The **`@type`** property identifies the subtype.

### AB Line (`ABLine`)

| Property | Type | Description |
| --- | --- | --- |
| `@type` | string | **`ABLine`** |
| `id` | string | Guidance line id. |
| `name` | string | Display name. |
| `lastModifiedTime` | datetime (ISO-8601) | Last processing or Operations Center modification time. |
| `archived` | boolean | Archived lines are not available in Setup Builder. |
| `tramOffset` | number | Spacing between tram lines. |
| `tramSpacing` | number | Tram lines relative to the main track (**0–20**). |
| `aPoint` | Point | Coordinate for point **A**. |
| `bPoint` | Point | Coordinate for point **B**; alternatively supply **heading** and this may be calculated. |
| `heading` | number | Radial heading (north = **0°**). Do not send both **bPoint** and **heading**. |
| `eastShift` | MeasurementAsDouble | East shift. |
| `northShift` | MeasurementAsDouble | North shift. |
| `spatialProjection` | SpatialProjection | Projection and elevation context. |
| `saveMethod` | string | Save method; see **dtABLineSaveMethod**. |
| `links` | array | HATEOAS links. |

### AB Curve (`ABCurve`)

Includes **`aPoint`**, **`bPoint`**, **`heading`**, shifts, **tram** fields, **`archived`**, optional **`shapes`** (when embedded), and **`links`**.

### Adaptive Curve (`AdaptiveCurve`)

Uses **`referencePoint`** as the origin for local coordinates, plus shifts, **`spatialProjection`**, **`archived`**, optional **`shapes`**, and **`links`**.

### Circle Track (`CircleTrack`)

Includes **`centerPoint`**, **`radialHeading`**, **`radiusOffset`**, **`radialShift`**, **tram** fields, **`archived`**, **`spatialProjection`**, and **`links`**.

### Shapes array (AB Curve / Adaptive Curve)

The **`shapes`** array holds geometry for curved paths when embedded. It is an array of **MultiPoint** objects; each **MultiPoint** is one continuous section of the track.

| Property | Type | Description |
| --- | --- | --- |
| `@type` | string | **`MultiPoint`** |
| `points` | Point[] | Vertices for that section. |

## GuidanceLine events (Data Subscription Service)

**GuidanceLine** events fire when guidance lines are **created**, **modified**, or **deleted**, or when related field operations affect them (for example merge, unmerge, archive, or unarchive per platform behavior). Subscribe through the **Data Subscription Service**; the client must specify an **`orgId`** filter. Other filters are optional. Filter values are **arrays**, so multiple **`orgId`** or **`guidanceLineId`** values may appear in one subscription.

Prefer **one organization per subscription**: you cannot change the **`orgId`** list after creation, and the subscription ends if the user loses access to any listed organization.

### Event statuses

| Status | Description |
| --- | --- |
| **ARCHIVED** | Guidance line is archived. |
| **AVAILABLE** | Guidance line is not archived and is available to use. |

### Event actions

| Action | Description |
| --- | --- |
| **CREATED** | Guidance line was created. |
| **DELETED** | Guidance line was deleted. |
| **MODIFIED** | Metadata update, archive/unarchive, or related field merge/unmerge/archive changes. |

### MetaDataFilter

| Filter | Required | Example | Description |
| --- | --- | --- | --- |
| **orgId** | Yes | `"123456"` | Organization associated with the guidance line. |
| **guidanceLineId** | No | UUID | Specific guidance line id. |
| **status** | No | `"AVAILABLE"` | **ARCHIVED** or **AVAILABLE**. |
| **action** | No | `"CREATED"` | **CREATED**, **DELETED**, or **MODIFIED**. |

## Authentication (OAuth 2.0)

John Deere **Precision Tech** APIs use **OAuth 2.0** with the **authorization code** grant. Create an application on [Developer.Deere.com](https://developer.deere.com/) for your **Application Id** and **Secret**. Register **Redirect URIs**; mismatched URIs can return **400 Bad Redirect**.

<Steps>
  <Step title="Discover endpoints and scopes">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Authorize">
    **GET** the authorize URL with `response_type=code`, scopes (**`ag1`** for read, **`ag3`** for create/update), **client_id**, **state**, and a **URL-encoded** `redirect_uri` registered on your application.
  </Step>
  <Step title="Exchange the code for tokens">
    **POST** `https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token` with `grant_type=authorization_code`. Access tokens expire after **12 hours**. Request **`offline_access`** for a **refresh token**.
  </Step>
  <Step title="Enable organization access">
    Call **GET** `/platform/organizations`. If a link has **`rel=connections`**, redirect the user so they can grant access. When complete, **`manage_connections`** appears instead.
  </Step>
  <Step title="Call Guidance Lines APIs">
    Send **`Authorization: Bearer {token}`** and **`Accept: application/vnd.deere.axiom.v3+json`**. For **POST**/**PUT**, send **`Content-Type: application/vnd.deere.axiom.v3+json`**.
  </Step>
</Steps>

### OAuth scopes for these endpoints

| Scope | Description |
| --- | --- |
| `ag1` | View locations — **GET** list, **GET** by id |
| `ag3` | Manage locations — **POST** (create AB Line), **PUT** (rename) |

Request only scopes your application needs. User permissions cap what the connection allows.

## Paging and links

List responses may use **CollectionPage** with **`total`**, **`links`**, and **`values`**. Use **`itemLimit`** (multiples of **10**, max **100**) and **`pageOffset`** where supported. Use **`showLinks=none`** or filtered **`showLinks`** to trim HATEOAS payloads.

**Note:** The guidance line **list** endpoint does **not** support **eTags** / **`x-deere-signature`**.

```text

```

<ParamField path="orgId" type="string" required>
  Organization id.
</ParamField>

<ParamField path="fieldId" type="string" required>
  Field id (GUID).
</ParamField>

**OAuth scope:** `ag3`

  --header 'Content-Type: application/vnd.deere.axiom.v3+json' \
