## Overview

The Quote Vendor Services API exposes quotation and purchase agreement operations for integrators working with **JDSC Quote**, including search, create and copy flows, equipment and trade-in management, PDFs, and delivery documents.

<CardGroup cols={2}>
  <Card title="Search & filter" icon="filter">
    Find active or deleted quotes with structured filter requests.
  </Card>
  <Card title="Create & copy" icon="copy">
    Create blank quotes, quotes from master templates, copies, and equipment copies.
  </Card>
  <Card title="Details & PDFs" icon="file-pdf">
    Retrieve quote, equipment, trade-in, agreement, and proposal PDFs.
  </Card>
  <Card title="Maintain quotes" icon="pen-to-square">
    Update name, status, customer, salesperson, store, and expiration; delete quotes or equipment.
  </Card>
</CardGroup>

## Authentication

John Deere Dealer Solutions APIs use **three-legged OAuth 2.0**. Register an application on [Developer.Deere.com](https://developer.deere.com) to obtain a **Client Key** and **Client Secret**, and configure at least one **Callback URL (Redirect URI)** on the OAuth profile for your application. If callback URLs do not sync correctly, users may see a `400 Bad Redirect` after authentication; contact John Deere support if that occurs.

<Steps>
  <Step title="Create an application on Developer.Deere.com">
    Your application receives a **Client Key** and **Client Secret**. You will use these credentials in the OAuth flow.
  </Step>
  <Step title="Add callback URLs">
    On the OAuth Profile page (under application Details), add one or more **Redirect URIs** and save. The `redirect_uri` query parameter must be URL-encoded when calling the authorization server.
  </Step>
  <Step title="Discover OAuth metadata">
    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
    This document lists authorization and token endpoints and supported scopes.
  </Step>
  <Step title="Acquire an authorization code">
    Redirect the user to the authorization server with `response_type=code`, requested **OAuth scopes** (see each endpoint; this API uses `openid` and `profile`), **client_id**, **state**, and **redirect_uri** (URL-encoded).
  </Step>
  <Step title="Exchange the code for an access token">
    POST to the token endpoint with grant type `authorization_code`, the authorization code, and **redirect_uri**. Access tokens expire after **12 hours**. Request **`offline_access`** if you need a refresh token.
  </Step>
  <Step title="Call the API">
    Send `Authorization: Bearer <access_token>` on every request. Set up **refresh token** rotation before expiry if you requested `offline_access`. Refresh tokens expire after **365 days** of non-use.
  </Step>
</Steps>

## Base URL

```
https://quote-api-sandbox.deere.com
```

API paths are rooted at `/om/cert/vendor` (for example, `POST /om/cert/vendor/v1/quotes/filter`).

## Endpoints

Each operation returns JSON (or PDF bytes for `*-pdf` routes) as described in the **OpenAPI** spec and API Playground. The examples below illustrate typical fields.

### Get Quotes

Filter quotes using search criteria. All request fields except `limit` and `offset` are required.

```
POST /om/cert/vendor/v1/quotes/filter
```

**OAuth scopes:** `openid`, `profile`

<ParamField body="(JSON)" type="object" required>
  Request body per API contract (see OpenAPI schema for example fields).
</ParamField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url 'https://quote-api-sandbox.deere.com/om/cert/vendor/v1/quotes/filter' \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --data '{"dealerId":"D-078901","fromDate":"2026-01-01","toDate":"2026-12-31","limit":50,"offset":0}'
```

</CodeGroup>

#### Response fields

<ResponseField name="quotes" type="array">
  Matching quote summaries (structure varies by environment).
</ResponseField>

<ResponseField name="totalCount" type="integer">
  Total matches for the filter (when paginating with `limit` / `offset`).
</ResponseField>

<Card title="Try it in the API Playground" icon="play" href="/POST /om/cert/vendor/v1/quotes/filter">
  Test this endpoint interactively with your access token.
</Card>

#### Sample response

```json
{
  "quotes": [
    {
      "quoteId": "QT-20260001",
      "name": "Spring equipment package",
      "status": "DRAFT"
    }
  ],
  "totalCount": 1
}
```

```

```

**OAuth scopes:** `openid`, `profile`

<ParamField body="(JSON)" type="object" required>
  Request body per API contract (see OpenAPI schema for example fields).
</ParamField>

  --url 'https://quote-api-sandbox.deere.com/om/cert/vendor/v1/quotes/quote-name' \

  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
