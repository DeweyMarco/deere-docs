## Overview

The Quote Data API is responsible for retrieving quote-related information.

## Authentication

The Quote Data API uses **three-legged OAuth 2.0** for authentication. This flow requires end-user consent and a John Deere sign-in.

<Steps>
  <Step title="Create an application on Developer.Deere.com">
    Register your application to receive a **Client Key** and **Client Secret**. All API requests are signed with these credentials.

    On the OAuth Profile page of your application (found in the Details section), add one or more **Callback URLs (Redirect URIs)**. Click **Add** to add more than one, then click **Save**.

    <Note>
      If callback URLs do not sync properly, users will receive a `400 Bad Redirect` error after authentication. Contact John Deere support if this occurs — the callback may need to be manually synced.
    </Note>
  </Step>
  <Step title="Call the well-known URL">
    Discover the authorization endpoint, token endpoint, and available scopes.

    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Acquire an authorization code">
    Redirect the user to the John Deere authorization server. The server will redirect back to your application with an authorization code after the user signs in and grants consent.

    Required query parameters: `response_type=code`, `client_id`, `redirect_uri` (URL-encoded), `scope`, and `state`.

    **OAuth Scopes:**

    | Scope | Permission | Description |
    |---|---|---|
    | `offline_access` | API authentication only | Request a refresh token |
  </Step>
  <Step title="Acquire an access token">
    Exchange the authorization code for an access token and refresh token.

    ```bash
    POST https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token
    ```

    **Headers:**

    ```
    Authorization: Basic <base64(clientKey:clientSecret)>
    Accept: application/json
    Content-Type: application/x-www-form-urlencoded
    ```

    **Body:**

    ```
    grant_type=authorization_code&code=<auth_code>&redirect_uri=<redirect_uri>
    ```

    **Response:**

    ```json
    {
      "token_type": "Bearer",
      "expires_in": 43200,
      "access_token": "eyJhbG[...]1LQ",
      "refresh_token": "eyJhbG[...]xYZ",
      "scope": "offline_access"
    }
    ```

    <Note>
      The access token expires after **12 hours**. Use the refresh token to obtain a new access token before expiry.
    </Note>
  </Step>
  <Step title="Call the API with your access token">
    Include the token in the `Authorization` header of every request.

    ```bash
    Authorization: Bearer <access_token>
    ```
  </Step>
  <Step title="Refresh the access token periodically">
    Before the access token expires, request a new one using the refresh token. The refresh token expires after **365 days** of inactivity.

    ```bash
    POST https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token
    ```

    **Body:**

    ```
    grant_type=refresh_token&refresh_token=<refresh_token>
    ```
  </Step>
</Steps>

## Endpoints

### Search Quote

Search for quotes using filter criteria such as dealer ID, customer name, status, or date range.

```
POST /om/cert/quotedata/api/v1/quote-data
```

<ParamField body="dealerId" type="string">
  Filter by dealer account identifier.
</ParamField>

<ParamField body="customerName" type="string">
  Filter by customer name (partial match supported).
</ParamField>

<ParamField body="status" type="string">
  Filter by quote status. One of: `DRAFT`, `ACTIVE`, `EXPIRED`, `DELETED`.
</ParamField>

<ParamField body="fromDate" type="string">
  Filter quotes created on or after this date (`YYYY-MM-DD`).
</ParamField>

<ParamField body="toDate" type="string">
  Filter quotes created on or before this date (`YYYY-MM-DD`).
</ParamField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url 'https://jdquote2-api-sandbox.deere.com/om/cert/quotedata/api/v1/quote-data' \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: */*' \
  --header 'Content-Type: application/json' \
  --data '{"dealerId":"D-078901","status":"ACTIVE","customerName":"Smith"}'
```

</CodeGroup>

#### Response fields

<ResponseField name="quotes" type="array">
  List of quotes matching the search criteria.
  <Expandable title="quotes">
    <ResponseField name="quoteId" type="string">
      Unique identifier for the quote.
    </ResponseField>
    <ResponseField name="dealerId" type="string">
      Dealer account identifier associated with the quote.
    </ResponseField>
    <ResponseField name="customerName" type="string">
      Name of the customer the quote is prepared for.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current status. One of: `DRAFT`, `ACTIVE`, `EXPIRED`, `DELETED`.
    </ResponseField>
    <ResponseField name="createdDate" type="string">
      ISO 8601 timestamp when the quote was created.
    </ResponseField>
    <ResponseField name="expirationDate" type="string">
      Date on which the quote expires.
    </ResponseField>
    <ResponseField name="totalAmount" type="number">
      Total quoted amount in USD.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="totalCount" type="integer">
  Total number of quotes matching the search criteria.
</ResponseField>

<Card title="Try it in the API Playground" icon="play" href="/POST /om/cert/quotedata/api/v1/quote-data">
  Test this endpoint interactively with your access token.
</Card>
