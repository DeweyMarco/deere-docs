## Overview

The Maintain Quotes API is designed to create and update the main aspects of a quotation. Use these endpoints to manage the full quote lifecycle — from initial creation to saving a finalized quote ready for the customer.

<CardGroup cols={2}>
  <Card title="Create & Copy Quotes" icon="file-plus">
    Create new quotes from scratch, from master templates, or by copying existing quotes.
  </Card>
  <Card title="Equipment Management" icon="tractor">
    Add and remove equipment line items sourced directly from the Dealer Business System (DBS).
  </Card>
  <Card title="Trade-In Data" icon="arrows-rotate">
    Capture and update trade-in equipment details to accurately reflect customer deals.
  </Card>
  <Card title="Master Quote Templates" icon="copy">
    Retrieve master quote templates and spin up new quotes from a standardized starting point.
  </Card>
</CardGroup>

## Authentication

The Maintain Quotes API uses **three-legged OAuth 2.0** for authentication. This flow requires end-user consent and a John Deere sign-in.

<Steps>
  <Step title="Create an application on Developer.Deere.com">
    Register your application to receive a **Client Key** and **Client Secret**. All API requests are signed with these credentials.

    On the OAuth Profile page of your application (found in the Details section), add one or more **Callback URLs (Redirect URIs)**. Click **Add** to add more than one, then click **Save**.

    <Note>
      If callback URLs do not sync properly, users will receive a `400 Bad Redirect` error after authentication. Contact John Deere support if this occurs — the callback may need to be manually synced.
    </Note>
  </Step>
  <Step title="Call the well-known URL">
    Discover the authorization endpoint, token endpoint, and available scopes.

    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Acquire an authorization code">
    Redirect the user to the John Deere authorization server. The server will redirect back to your application with an authorization code after the user signs in and grants consent.

    Required query parameters: `response_type=code`, `client_id`, `redirect_uri` (URL-encoded), `scope`, and `state`.

    **OAuth Scopes:**

    | Scope | Permission | Description |
    |---|---|---|
    | `axiom` | API access | Required for all Maintain Quotes endpoints |
    | `offline_access` | API authentication only | Request a refresh token |
  </Step>
  <Step title="Acquire an access token">
    Exchange the authorization code for an access token and refresh token.

    ```bash
    POST https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token
    ```

    **Headers:**

    ```
    Authorization: Basic <base64(clientKey:clientSecret)>
    Accept: application/json
    Content-Type: application/x-www-form-urlencoded
    ```

    **Body:**

    ```
    grant_type=authorization_code&code=<auth_code>&redirect_uri=<redirect_uri>
    ```

    **Response:**

    ```json
    {
      "token_type": "Bearer",
      "expires_in": 43200,
      "access_token": "eyJhbG[...]1LQ",
      "refresh_token": "eyJhbG[...]xYZ",
      "scope": "axiom offline_access"
    }
    ```

    <Note>
      The access token expires after **12 hours**. Use the refresh token to obtain a new access token before expiry.
    </Note>
  </Step>
  <Step title="Call the API with your access token">
    Include the token in the `Authorization` header of every request.

    ```bash
    Authorization: Bearer <access_token>
    ```
  </Step>
  <Step title="Refresh the access token periodically">
    Before the access token expires, request a new one using the refresh token. The refresh token expires after **365 days** of inactivity.

    ```bash
    POST https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token
    ```

    **Body:**

    ```
    grant_type=refresh_token&refresh_token=<refresh_token>
    ```
  </Step>
</Steps>

## Endpoints

### Get Master Quotes

Returns the list of master quote templates for a given dealer.

```
POST /api/v1/dealers/{dealerId}/quotes
```

**OAuth Scope Required:** `axiom`

<ParamField path="dealerId" type="string" required>
  The unique dealer account identifier.
</ParamField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url 'https://jdquote2-api-sandbox.deere.com/api/v1/dealers/D-078901/quotes' \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: */*'
```

</CodeGroup>

#### Response fields

<ResponseField name="masterQuotes" type="array">
  List of master quote templates for the dealer.
  <Expandable title="masterQuotes">
    <ResponseField name="masterQuoteId" type="string">
      Unique identifier for the master quote.
    </ResponseField>
    <ResponseField name="name" type="string">
      Descriptive name for the master quote template.
    </ResponseField>
    <ResponseField name="dealerId" type="string">
      Dealer account identifier that owns this master quote.
    </ResponseField>
    <ResponseField name="createdDate" type="string">
      ISO 8601 timestamp when the master quote was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<Card title="Try it in the API Playground" icon="play" href="/POST /api/v1/dealers/{dealerId}/quotes">
  Test this endpoint interactively with your access token.
</Card>
