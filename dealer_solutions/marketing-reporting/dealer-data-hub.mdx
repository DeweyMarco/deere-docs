## Authentication

The John Deere Dealer Solutions API endpoints use the **client credentials** grant type for OAuth 2.0 authentication.

<Steps>
  <Step title="Create an application on Developer.Deere.com">
    When you create an application, it is assigned a **Client Key** and **Client Secret**. You sign API requests with these credentials.

    In the **Security** section of your application (under application details), add at least one **Callback URL (Redirect URI)**. While a redirect URI may not be required for the client credentials flow, add one as part of the registration process. You may use a placeholder such as `https://localhost:9090/callback` if needed.
  </Step>
  <Step title="Call the OAuth 2 well-known URL">
    Discover the authorization and token endpoints and the scopes your integration can request.

    ```bash
    GET https://signin.johndeere.com/oauth2/ausabzzga86EIeR4l1t7/.well-known/oauth-authorization-server
    ```

    OAuth scopes required for each endpoint are documented with the API. Not every API requires a scope; confirm the scope for Dealer Data Hub in your technical documentation and application configuration.
  </Step>
  <Step title="Acquire an access token">
    POST to the token endpoint using the `client_credentials` grant type. Place your Client Key and Secret in the HTTP `Authorization` header using **HTTP Basic Auth** — not in the request body.

    ```bash
    POST https://signin.johndeere.com/oauth2/ausabzzga86EIeR4l1t7/v1/token
    ```

    **Headers:**

    ```
    Authorization: Basic <base64(clientKey:clientSecret)>
    Accept: application/json
    Content-Type: application/x-www-form-urlencoded
    ```

    **Body:**

    ```
    grant_type=client_credentials&scope=<your_scope>
    ```

    **Response:**

    ```json
    {
      "token_type": "Bearer",
      "expires_in": 43200,
      "access_token": "eyJhbG[...]1LQ"
    }
    ```

    <Note>
      Access tokens expire; refresh by requesting a new token before expiry. The `expires_in` value (seconds) returned by the token endpoint is authoritative for your client.
    </Note>
  </Step>
  <Step title="Call the API with your access token">
    Include the token in the `Authorization` header of every request to Dealer Data Hub.

    ```bash
    Authorization: Bearer <access_token>
    ```
  </Step>
</Steps>

## Endpoints

### Search dealer by employee XID

Search for dealer data associated with an **employee XID**. Use this when your integration keys off employee identity to resolve dealer context.

```
POST /api/search/xid/{xid}
```

**OAuth scope:** Use the scope configured for Dealer Data Hub on your Developer.Deere.com application (this documentation uses the `ddh` scope in the API reference as an example — replace with your assigned scope if different).

<ParamField path="xid" type="string" required>
  Employee XID used to search dealer records.
</ParamField>

<ParamField body="includeInactive" type="boolean">
  Optional. When supported, whether to include inactive associations in the result set.
</ParamField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url 'https://dealer-data-hub-sandbox.deere.com/api/search/xid/emp-xid-001234567890' \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.ddh.v2+json' \
  --header 'Content-Type: application/vnd.deere.ddh.v2+json' \
  --data '{}'
```

</CodeGroup>

#### Response fields

<ResponseField name="dealers" type="array">
  Dealer records or aggregates returned for the XID search. Exact fields follow Dealer Data Hub v2.
</ResponseField>

<ResponseField name="meta" type="object">
  Optional result metadata (for example counts or pagination hints).
</ResponseField>

<Card title="Try it in the API Playground" icon="play" href="/POST /api/search/xid/{xid}">
  Test this endpoint interactively with your access token.
</Card>

#### Sample response

```json
{
  "dealers": [
    {
      "dealerId": "D-078901",
      "dealerName": "Example Ag & Turf",
      "employeeXid": "emp-xid-001234567890"
    }
  ],
  "meta": {
    "resultCount": 1
  }
}
```

### Dealer 360-degree view

Provide a **360-degree view** of dealer data through various digital threads (activities, analytics, and related context).

```
POST /api/search/dealers
```

**OAuth scope:** Use the scope configured for Dealer Data Hub on your Developer.Deere.com application (see note above).

<ParamField body="dealerId" type="string">
  Optional dealer identifier to scope the 360-degree view when applicable.
</ParamField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url 'https://dealer-data-hub-sandbox.deere.com/api/search/dealers' \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/vnd.deere.ddh.v2+json' \
  --header 'Content-Type: application/vnd.deere.ddh.v2+json' \
  --data '{"dealerId":"D-078901"}'
```

</CodeGroup>

#### Response fields

<ResponseField name="dealers" type="array">
  Dealer-level payloads for the 360-degree view. Structure aligns with Dealer Data Hub digital threads.
</ResponseField>

<ResponseField name="meta" type="object">
  Optional result metadata.
</ResponseField>

<Card title="Try it in the API Playground" icon="play" href="/POST /api/search/dealers">
  Test this endpoint interactively with your access token.
</Card>

#### Sample response

```json
{
  "dealers": [
    {
      "dealerId": "D-078901",
      "dealerName": "Example Ag & Turf",
      "digitalThreads": ["sales", "service", "parts"]
    }
  ],
  "meta": {
    "resultCount": 1
  }
}
```
