## Overview

The Market Share API is a REST API that provides U.S. and Canada dealer market share data for Agriculture, Construction & Forestry (C&F), and Construction & Compact Equipment (CCE). It aligns with data provided through the Global Market Share (GMS) application, including both **By Store** and **By County** structures.

Using market share data can help your organization track performance against sales and market share goals and support insight into potential pay-for-performance incentives.

## Authentication

The Market Share API uses the **client credentials** OAuth 2.0 grant type.

<Steps>
  <Step title="Create an application on Developer.Deere.com">
    Register your application to receive a **Client Key** and **Client Secret**. All API requests are signed with these credentials.

    In the Security section of your application, add at least one Callback URL (Redirect URI). You may use a placeholder such as `https://localhost:9090/callback` if a redirect URI is not required for your flow.
  </Step>
  <Step title="Call the well-known URL">
    Discover the authorization and token endpoints, as well as the available scopes.

    ```bash
    GET https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/.well-known/oauth-authorization-server
    ```
  </Step>
  <Step title="Acquire an access token">
    POST to the token endpoint using the `client_credentials` grant type. Place your Client Key and Secret in the HTTP `Authorization` header using HTTP Basic Auth — **not** in the request body.

    ```bash
    POST https://signin.johndeere.com/oauth2/aus78tnlaysMraFhC1t7/v1/token
    ```

    **Headers:**

    ```
    Authorization: Basic <base64(clientKey:clientSecret)>
    Accept: application/json
    Content-Type: application/x-www-form-urlencoded
    ```

    **Body:**

    ```
    grant_type=client_credentials&scope=axiom
    ```

    **Response:**

    ```json
    {
      "token_type": "Bearer",
      "expires_in": 43200,
      "access_token": "eyJhbG[...]1LQ",
      "scope": "axiom"
    }
    ```

    <Note>
      The access token expires after **12 hours**. Refresh it by repeating this step before expiry.
    </Note>
  </Step>
  <Step title="Call the API with your access token">
    Include the token in the `Authorization` header of every request.

    ```bash
    Authorization: Bearer <access_token>
    ```
  </Step>
</Steps>

## Endpoints

### Get AEM industry average dollar data

Returns the Association for Equipment Manufacturers (AEM) list of average dollar values for each unique machine and size class combination. For North America, these values are typically updated annually and made available in February.

```
GET /mrkt/sandbox/drx-api/avgDollarReport/{dealerId}
```

**OAuth Scope Required:** `axiom`

<ParamField path="dealerId" type="string" required>
  Dealer identifier for which to retrieve AEM average dollar data.
</ParamField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url 'https://drx-api-sandbox.deere.com/mrkt/sandbox/drx-api/avgDollarReport/D-078901' \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/json'
```

</CodeGroup>

#### Response fields

<ResponseField name="dealerId" type="string">
  Dealer identifier for the report.
</ResponseField>

<ResponseField name="rows" type="array">
  AEM average dollar rows for each unique machine and size class combination.
  <Expandable title="rows">
    <ResponseField name="machine" type="string">
      Machine category or model family identifier.
    </ResponseField>
    <ResponseField name="sizeClass" type="string">
      Equipment size class.
    </ResponseField>
    <ResponseField name="averageDollarValue" type="number">
      AEM average dollar value for this combination.
    </ResponseField>
    <ResponseField name="currency" type="string">
      ISO 4217 currency code (e.g. `USD`).
    </ResponseField>
    <ResponseField name="effectiveYear" type="integer">
      Year the AEM values apply to.
    </ResponseField>
  </Expandable>
</ResponseField>

### Retrieve market share data by county

Returns the same data as the Global Market Share (GMS) application’s “Raw Data (by County)” report (“ByCounty” tab): dealer and industry sales for each unique Machine, Size Class, County, and Activity Month combination.

```
POST /mrkt/sandbox/drx-api/rawDataReport/byCounty
```

**OAuth Scope Required:** `axiom`

<ParamField body="dealerId" type="string">
  Dealer identifier requesting the report.
</ParamField>

<ParamField body="reportingPeriodStart" type="string">
  Start of the activity month range (inclusive), ISO 8601 date.
</ParamField>

<ParamField body="reportingPeriodEnd" type="string">
  End of the activity month range (inclusive), ISO 8601 date.
</ParamField>

<ParamField body="segments" type="array">
  Optional market segments to include (e.g. Ag, C&F, CCE).
</ParamField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url 'https://drx-api-sandbox.deere.com/mrkt/sandbox/drx-api/rawDataReport/byCounty' \
  --header 'Authorization: Bearer <access_token>' \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --data '{"dealerId":"D-078901","reportingPeriodStart":"2025-01-01","reportingPeriodEnd":"2025-12-31","segments":["AG","CF","CCE"]}'
```

</CodeGroup>

#### Response fields

<ResponseField name="rows" type="array">
  Report rows aligned with the GMS By County raw data structure. Exact fields may vary by segment and reporting period.
</ResponseField>

<ResponseField name="rowCount" type="integer">
  Number of rows returned in this response.
</ResponseField>

</Card>
