## Overview

The Dealer Data Feed (DDF) CoreMedia APIs enable John Deere Dealers to effortlessly access and synchronize comprehensive product information from the John Deere New Websites using RESTful APIs.

This integration ensures that dealer websites consistently display the most current product and marketing details available on Deere.com. By utilizing these APIs, dealers can keep their digital presence accurate and up-to-date, improving both operational efficiency and customer experience.

## Authentication

All DDF API requests require a valid OAuth 2.0 bearer token issued by John Deere's authorization server.

```bash
Authorization: Bearer <access_token>
```

<Note>
  Contact your John Deere partner representative to obtain API credentials and access to the sandbox environment.
</Note>

## Endpoints

### Retrieve all product categories

Fetch all product categories with optional hierarchical children (up to 3 levels deep).

```
GET /caas/v1/ddf/categories
```

<ParamField query="locale" type="string">
  Language and region code for localized content. Examples: `en_US`, `de_DE`, `fr_FR`, `en_IE`.
</ParamField>

<ParamField query="includeChildren" type="boolean">
  When `true`, includes child categories in the response hierarchy (up to 3 levels deep).
</ParamField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url 'https://developer.deere.com/caas/v1/ddf/categories?locale=en_US&includeChildren=true' \
  --header 'Authorization: Bearer <access_token>'
```

</CodeGroup>

#### Response fields

<ResponseField name="taxonomies" type="object">
  The root object containing category data.
  <Expandable title="taxonomies">
    <ResponseField name="categories" type="array">
      List of top-level product categories.
      <Expandable title="categories[]">
        <ResponseField name="id" type="string">
          Unique identifier for the category.
        </ResponseField>
        <ResponseField name="name" type="string">
          Human-readable category name.
        </ResponseField>
        <ResponseField name="baseCode" type="string">
          Internal base code for the category (e.g. `WG_products-and-solutions`).
        </ResponseField>
        <ResponseField name="canonicalPath" type="string">
          Full canonical URL to the category on Deere.com.
        </ResponseField>
        <ResponseField name="externalPath" type="string">
          Relative path used externally.
        </ResponseField>
        <ResponseField name="locale" type="string">
          Locale the content is returned in.
        </ResponseField>
        <ResponseField name="isProduct" type="boolean">
          Whether this category represents a product. Present on child nodes only.
        </ResponseField>
        <ResponseField name="children" type="array">
          Nested child categories. Present only when `includeChildren=true`. Same shape as parent, up to 3 levels deep.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Retrieve product information

Retrieve detailed product data for a specific category, including specifications, features, and benefits.

```
GET /caas/v1/ddf/products
```

<ParamField query="categoryId" type="string" required>
  The category ID to retrieve product information for.
</ParamField>

<ParamField query="locale" type="string">
  Language and region code for localized content. Example: `en_US`.
</ParamField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url 'https://developer.deere.com/caas/v1/ddf/products?categoryId=183684&locale=en_US' \
  --header 'Authorization: Bearer <access_token>'
```

</CodeGroup>
